# Goal

Build the customization engine accessible from the side menu. Users manage their own animal types, breeds (with optional gestation day overrides), care templates, and event templates. Starter playbooks are seeded during onboarding based on selected species. This is what makes the app feel personal — it adapts to each homesteader's specific animals and routines.

**Architecture notes:**
- `IAnimalTypeService.ts` (interface) + `AnimalTypeService.ts` (implementation) — Interface + Implementation pattern
- `AnimalTypeService` calls `@react-native-firebase/firestore` directly via homestead subcollection: `this.homesteadRef.collection('animalType')`
- Uses `onSnapshot` for the animal types list (real-time updates)
- Store teardown pattern: stores hold unsubscribe internally with `subscribe()`, `teardown()`, `clear()`
- `seedStarterPlaybook` uses `firestore().batch()` for atomic writes, takes `homesteadId` parameter
- Types (`AnimalType`, `Breed`, `CareTemplate`, `EventTemplate`) imported from `schema/animalType/` — not re-defined in the service (D9)
- No `userId` in queries or documents — homestead scoping replaces user scoping
- AdminObject uses Firebase Timestamps (`Tstamp`)
- Service exported from `Bootstrap.ts` as `bsAnimalTypeService`

## Firestore Schemas

### Animal Types — `/homestead/{homesteadId}/animalType/{typeId}` (homestead subcollection)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | Yes | Document ID |
| `admin` | `AdminObject` | Yes | `deleted`, `updated_at` (Tstamp), `created_at` (Tstamp) |
| `name` | `string` | Yes | Species display name (e.g., "Chicken", "Goat") |
| `colors` | `string[]` | Yes | Available color options for animals of this type |

### Breeds — `/homestead/{homesteadId}/animalType/{typeId}/breed/{breedId}` (subcollection)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | Yes | Document ID |
| `admin` | `AdminObject` | Yes | Metadata |
| `name` | `string` | Yes | Breed name (e.g., "Nubian", "Rhode Island Red") |
| `gestationDays` | `number` | No | Override species-level gestation period for this breed |

### Care Templates — `/homestead/{homesteadId}/animalType/{typeId}/careTemplate/{templateId}` (subcollection)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | Yes | Document ID |
| `admin` | `AdminObject` | Yes | Metadata |
| `name` | `string` | Yes | Template name (e.g., "Quarterly Deworming") |
| `type` | `"careRecurring" \| "careSingle"` | Yes | Whether the care recurs |
| `cycle` | `number` | No | Recurrence interval in days. Required if `type` is `"careRecurring"` |
| `contactName` | `string` | No | Default provider name |
| `contactPhone` | `string` | No | Default provider phone |

### Event Templates — `/homestead/{homesteadId}/animalType/{typeId}/eventTemplate/{templateId}` (subcollection)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | Yes | Document ID |
| `admin` | `AdminObject` | Yes | Metadata |
| `name` | `string` | Yes | Template name |

## Edit Animal Type Screen

### `EditAnimalTypeController.ts`
```typescript
import { useState } from 'react'
import { bsAnimalTypeService } from '../../../Bootstrap'

export function useEditAnimalTypeController(navigation: any, route: any) {
  const { animalTypeId } = route.params ?? {}
  const [name, setName] = useState('')
  const [colors, setColors] = useState<string[]>([])
  const [newColor, setNewColor] = useState('')
  const [loading, setLoading] = useState(false)
  const [isEditing] = useState(!!animalTypeId)

  // Load existing data if editing
  useState(() => {
    if (animalTypeId) {
      bsAnimalTypeService.fetchAnimalType(animalTypeId).then(type => {
        if (type) {
          setName(type.name)
          setColors(type.colors)
        }
      })
    }
  })

  const addColor = () => {
    if (newColor.trim() && !colors.includes(newColor.trim())) {
      setColors([...colors, newColor.trim()])
      setNewColor('')
    }
  }

  const removeColor = (color: string) => setColors(colors.filter(c => c !== color))

  const save = async () => {
    if (!name.trim()) return
    setLoading(true)
    if (isEditing) {
      await bsAnimalTypeService.updateAnimalType({
        id: animalTypeId, admin: {} as any, name, colors,
      })
    } else {
      await bsAnimalTypeService.createAnimalType({
        id: '', admin: {} as any, name, colors,
      })
    }
    setLoading(false)
    navigation.goBack()
  }

  return { name, setName, colors, newColor, setNewColor, addColor, removeColor, loading, save, isEditing }
}
```

### Implementation (`apps/mobile/src/feature/customization/service/AnimalTypeService.ts`)

The service calls `@react-native-firebase/firestore` directly via homestead subcollection. Types are imported from schema — not re-defined here (D9). Uses `onSnapshot` for the animal types list. Uses `firestore().batch()` for seeding starter playbooks.

```typescript
import firestore, { FirebaseFirestoreTypes } from '@react-native-firebase/firestore'
import { IResult, SuccessResult, ErrorResult } from '../../../util/Result'
import { adminObject_default } from '../../../schema/object/AdminObject'
import Log from '../../../library/log/Log'
import { AnimalType, Breed, CareTemplate, EventTemplate } from '../../../schema/animalType/AnimalType'
import { IAnimalTypeService } from './IAnimalTypeService'

export default class AnimalTypeService implements IAnimalTypeService {
  private homesteadRef: FirebaseFirestoreTypes.DocumentReference

  constructor(homesteadId: string) {
    this.homesteadRef = firestore().collection('homestead').doc(homesteadId)
  }

  /**
   * Real-time subscription to animal types via onSnapshot.
   * Returns an unsubscribe function.
   */
  subscribeToAnimalTypes(callback: (types: AnimalType[]) => void): () => void {
    return this.homesteadRef
      .collection('animalType')
      .where('admin.deleted', '==', false)
      .onSnapshot(
        snapshot => {
          const types = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() } as AnimalType))
          callback(types)
        },
        error => {
          Log.error('AnimalTypeService', 'subscribeToAnimalTypes error: ' + error)
          callback([])
        },
      )
  }

  async fetchAnimalType(animalTypeId: string): Promise<AnimalType | null> {
    try {
      const doc = await this.homesteadRef.collection('animalType').doc(animalTypeId).get()
      if (!doc.exists) return null
      return { id: doc.id, ...doc.data() } as AnimalType
    } catch (error) {
      Log.error('AnimalTypeService', 'fetchAnimalType failed: ' + error)
      return null
    }
  }

  async createAnimalType(type: AnimalType): Promise<IResult> {
    try {
      const docRef = this.homesteadRef.collection('animalType').doc()
      await docRef.set({
        ...type,
        id: docRef.id,
        admin: adminObject_default(),
      })
      return SuccessResult
    } catch (error) {
      Log.error('AnimalTypeService', 'createAnimalType failed: ' + error)
      return ErrorResult('Failed to create animal type: ' + error)
    }
  }

  async updateAnimalType(type: AnimalType): Promise<IResult> {
    try {
      await this.homesteadRef.collection('animalType').doc(type.id).update({
        name: type.name,
        colors: type.colors,
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })
      return SuccessResult
    } catch (error) {
      Log.error('AnimalTypeService', 'updateAnimalType failed: ' + error)
      return ErrorResult('Failed to update animal type: ' + error)
    }
  }

  async fetchBreeds(animalTypeId: string): Promise<Breed[]> {
    try {
      const snapshot = await this.homesteadRef
        .collection('animalType').doc(animalTypeId)
        .collection('breed')
        .where('admin.deleted', '==', false)
        .get()
      return snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() } as Breed))
    } catch (error) {
      Log.error('AnimalTypeService', 'fetchBreeds failed: ' + error)
      return []
    }
  }

  async createBreed(animalTypeId: string, breed: Breed): Promise<IResult> {
    try {
      const docRef = this.homesteadRef
        .collection('animalType').doc(animalTypeId)
        .collection('breed').doc()
      await docRef.set({
        ...breed,
        id: docRef.id,
        admin: adminObject_default(),
      })
      return SuccessResult
    } catch (error) {
      Log.error('AnimalTypeService', 'createBreed failed: ' + error)
      return ErrorResult('Failed to create breed: ' + error)
    }
  }

  async fetchCareTemplates(animalTypeId: string): Promise<CareTemplate[]> {
    try {
      const snapshot = await this.homesteadRef
        .collection('animalType').doc(animalTypeId)
        .collection('careTemplate')
        .where('admin.deleted', '==', false)
        .get()
      return snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() } as CareTemplate))
    } catch (error) {
      Log.error('AnimalTypeService', 'fetchCareTemplates failed: ' + error)
      return []
    }
  }

  async createCareTemplate(animalTypeId: string, template: CareTemplate): Promise<IResult> {
    try {
      const docRef = this.homesteadRef
        .collection('animalType').doc(animalTypeId)
        .collection('careTemplate').doc()
      await docRef.set({
        ...template,
        id: docRef.id,
        admin: adminObject_default(),
      })
      return SuccessResult
    } catch (error) {
      Log.error('AnimalTypeService', 'createCareTemplate failed: ' + error)
      return ErrorResult('Failed to create care template: ' + error)
    }
  }

  async fetchEventTemplates(animalTypeId: string): Promise<EventTemplate[]> {
    try {
      const snapshot = await this.homesteadRef
        .collection('animalType').doc(animalTypeId)
        .collection('eventTemplate')
        .where('admin.deleted', '==', false)
        .get()
      return snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() } as EventTemplate))
    } catch (error) {
      Log.error('AnimalTypeService', 'fetchEventTemplates failed: ' + error)
      return []
    }
  }

  /**
   * Seeds starter playbook data during onboarding.
   * Uses firestore().batch() directly for atomic write of all documents.
   * Takes homesteadId to scope data under the correct homestead.
   */
  async seedStarterPlaybook(homesteadId: string, species: string): Promise<void> {
    const { STARTER_PLAYBOOKS } = require('../data/StarterPlaybooks')
    const playbook = STARTER_PLAYBOOKS[species]
    if (!playbook) return

    const seedRef = firestore().collection('homestead').doc(homesteadId)

    try {
      const batch = firestore().batch()

      const typeRef = seedRef.collection('animalType').doc()
      batch.set(typeRef, {
        id: typeRef.id,
        admin: adminObject_default(),
        name: species,
        colors: playbook.colors,
      })

      for (const breed of playbook.breeds) {
        const breedRef = typeRef.collection('breed').doc()
        batch.set(breedRef, {
          id: breedRef.id,
          admin: adminObject_default(),
          name: breed.name,
          ...(breed.gestationDays ? { gestationDays: breed.gestationDays } : {}),
        })
      }

      for (const template of playbook.careTemplates) {
        const templateRef = typeRef.collection('careTemplate').doc()
        batch.set(templateRef, {
          id: templateRef.id,
          admin: adminObject_default(),
          name: template.name,
          type: template.type,
          cycle: template.cycle || undefined,
        })
      }

      await batch.commit()
    } catch (error) {
      Log.error('AnimalTypeService', 'seedStarterPlaybook failed for ' + species + ': ' + error)
    }
  }
}
```

# Notes

- Starter playbooks are seeded once during onboarding — the data is copied into the homestead's Firestore subcollections
- `seedStarterPlaybook` takes `homesteadId` parameter (not `userId`) to scope data correctly
- Template updates do NOT retroactively change existing care events — events are independent copies
- Breeds can optionally override the species default gestation days
- Colors array on AnimalType is used when creating/editing an animal (color dropdown options)
- Users can create entirely custom animal types beyond the defaults (e.g., "Bees", "Fish")
- The customization engine is what makes the app feel like "my farm app" vs "a generic template"
- **onSnapshot** is used for the animal types list — when the user adds/edits/deletes a type, the home screen updates in real-time without manual refresh
- Store teardown pattern: stores hold unsubscribe internally with `subscribe()`, `teardown()`, `clear()`
- Subcollections (breed, careTemplate, eventTemplate) are fetched on demand when viewing an animal type detail — no onSnapshot needed for these
- All data scoped under `/homestead/{homesteadId}/animalType/` — no `userId` field needed
- Service implements `IAnimalTypeService` interface — Interface + Implementation pattern
- `seedStarterPlaybook` uses `firestore().batch()` for atomic writes — all breeds and templates for a species are written in a single batch commit
- AdminObject timestamps use Firebase `Tstamp` (server timestamps on create, updated on mutation)
