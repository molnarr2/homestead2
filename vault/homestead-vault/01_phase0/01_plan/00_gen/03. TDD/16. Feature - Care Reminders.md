# Goal

Build the Care tab with cross-animal care schedule, care event CRUD, recurring care auto-creation on completion, and due date status coloring. Care reminders are the backbone of daily app usage — they turn the app into an indispensable daily checklist.

# Instructions

## Architecture Notes

1. `CareService` calls `@react-native-firebase/firestore` directly.
2. `admin.created_at` and `admin.updated_at` are `Tstamp` (Firebase Timestamp | FieldValue). Use `adminObject_default()` and `adminObject_updateLastUpdated()`.
3. `CareService.subscribe()` uses `this.homesteadRef.collection('careEvent').where(...).onSnapshot(...)` so completing a care event instantly updates the list across all screens without manual refetch.
4. `completeCareEvent` uses `firestore().batch()` for atomic completion + next-recurring-event creation.
5. Service location: `apps/mobile/src/feature/care/`.
6. Bootstrap export: `bsCareService: ICareService` from `apps/mobile/src/Bootstrap.ts`.

## Care Service (`apps/mobile/src/feature/care/service/`)

### `ICareService.ts`
```typescript
import { IResult } from '../../../util/Result'
import { CareEvent } from '../../../schema/care/CareEvent'

export default interface ICareService {
  subscribe(onUpdate: (events: CareEvent[]) => void): () => void
  createCareEvent(event: CareEvent): Promise<IResult>
  completeCareEvent(event: CareEvent): Promise<IResult>
}
```

### `CareService.ts`
```typescript
import firestore from '@react-native-firebase/firestore'
import { IResult, SuccessResult, ErrorResult } from '../../../util/Result'
import { CareEvent } from '../../../schema/care/CareEvent'
import { dateToTstamp } from '../../../schema/type/Tstamp'
import { adminObject_default, adminObject_updateLastUpdated } from '../../../schema/object/AdminObject'
import Log from '../../../library/log/Log'

export default class CareService implements ICareService {

  subscribe(onUpdate: (events: CareEvent[]) => void): () => void {
    return this.homesteadRef
      .collection('careEvent')
      .where('admin.deleted', '==', false)
      .orderBy('dueDate')
      .onSnapshot(
        snapshot => {
          const events: CareEvent[] = snapshot.docs.map(doc => ({
            ...doc.data(),
            id: doc.id,
          } as CareEvent))
          onUpdate(events)
        },
        error => {
          Log.error('CareService', 'subscribe error: ' + error.message)
        }
      )
  }

  async createCareEvent(event: CareEvent): Promise<IResult> {
    try {
      const docRef = this.homesteadRef.collection('careEvent').doc()
      event.id = docRef.id
      event.admin = adminObject_default()
      event.createdNextRecurringEvent = false
      await docRef.set(event)
      return SuccessResult
    } catch (error: any) {
      Log.error('CareService', 'createCareEvent error: ' + error.message)
      return ErrorResult(error.message)
    }
  }

  async completeCareEvent(event: CareEvent): Promise<IResult> {
    try {
      const batch = firestore().batch()
      const completedDate = dateToTstamp(new Date())

      // 1. Update the completed event
      const eventRef = this.homesteadRef.collection('careEvent').doc(event.id)
      batch.update(eventRef, {
        completedDate,
        createdNextRecurringEvent: event.type === 'careRecurring',
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })

      // 2. If recurring, create the next care event
      if (event.type === 'careRecurring' && !event.createdNextRecurringEvent) {
        const nextDueDate = calculateNextDueDate(completedDate, event.cycle)
        const nextEventRef = this.homesteadRef.collection('careEvent').doc()
        const nextEvent: CareEvent = {
          id: nextEventRef.id,
          animalId: event.animalId,
          templateId: event.templateId,
          name: event.name,
          type: event.type,
          cycle: event.cycle,
          dueDate: nextDueDate,
          completedDate: null,
          contactName: event.contactName,
          contactPhone: event.contactPhone,
          notes: event.notes,
          photoStorageRef: '',
          photoUrl: '',
          createdNextRecurringEvent: false,
          admin: adminObject_default(),
        }
        batch.set(nextEventRef, nextEvent)
      }

      // 3. Commit atomically
      await batch.commit()
      return SuccessResult
    } catch (error: any) {
      Log.error('CareService', 'completeCareEvent error: ' + error.message)
      return ErrorResult(error.message)
    }
  }
}
```

**Critical business logic — `completeCareEvent`:**
- Uses `firestore().batch()` directly for atomicity — both the completion update and the next event creation succeed or fail together.
- Next due date is calculated from **completion date**, NOT original due date — prevents cascading drift when completed late.
- One-time events (`careSingle`) don't create next occurrences.
- The `createdNextRecurringEvent` flag prevents duplicate next-event creation on retry.
- The Cloud Function `onCareEventComplete` trigger is a safety net for this logic.

## CareStore (Zustand)

```typescript
import { create } from 'zustand'
import { CareEvent } from '../../schema/care/CareEvent'
import { bsCareService } from '../../Bootstrap'

interface CareStoreState {
  careEvents: CareEvent[]
  loading: boolean
  unsubscribe: (() => void) | null

  subscribe: () => void
  teardown: () => void
  clear: () => void
}

export const useCareStore = create<CareStoreState>((set, get) => ({
  careEvents: [],
  loading: true,
  unsubscribe: null,

  subscribe: () => {
    get().teardown()
    set({ loading: true })
    const unsub = bsCareService.subscribe((careEvents) => {
      set({ careEvents, loading: false })
    })
    set({ unsubscribe: unsub })
  },

  teardown: () => {
    const unsub = get().unsubscribe
    if (unsub) unsub()
    set({ unsubscribe: null })
  },

  clear: () => {
    get().teardown()
    set({ careEvents: [], loading: true, unsubscribe: null })
  },
}))
```

The real-time `onSnapshot` listener pushes updates to the store. Completing a care event triggers a Firestore write, which triggers the `onSnapshot` callback, which updates the store — all screens update instantly.

## Care List Screen (Care Tab)

### `CareListController.ts`
```typescript
export function useCareListController(navigation) {
  const { careEvents, loading } = useCareStore()
  const { animals } = useAnimalStore()
  const [filterAnimalId, setFilterAnimalId] = useState<string | null>(null)

  const incompleteEvents = careEvents.filter(e => !e.completedDate)

  const filteredEvents = filterAnimalId
    ? incompleteEvents.filter(e => e.animalId === filterAnimalId)
    : incompleteEvents

  const overdue = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'OVERDUE')
  const dueToday = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'DUE_TODAY')
  const upcoming = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'UPCOMING')
  const future = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'FUTURE')

  const getAnimalName = (animalId: string) => animals.find(a => a.id === animalId)?.name ?? ''

  const onEventPress = (eventId: string) => navigation.navigate('CareEventDetail', { eventId })
  const onCreateEvent = () => navigation.navigate('CreateCareEvent', { animalId: filterAnimalId ?? '' })

  const onComplete = async (event: CareEvent) => {
    await bsCareService.completeCareEvent(event)
  }

  return {
    overdue, dueToday, upcoming, future, loading,
    animals, filterAnimalId, setFilterAnimalId, getAnimalName,
    onEventPress, onCreateEvent, onComplete,
  }
}
```

**Controller notes:**
- Store already has real-time data from `onSnapshot` — no `useFocusEffect` / `fetchCareEvents()` call needed.
- Real-time listener makes pull-to-refresh unnecessary (can still be kept for UX familiarity).
- `onComplete` calls `bsCareService.completeCareEvent()` directly. The store updates automatically via the `onSnapshot` listener.

export function useCareEventDetailController(navigation, route) {
  const { eventId } = route.params
  const { careEvents } = useCareStore()
  const { animals } = useAnimalStore()
  const event = careEvents.find(e => e.id === eventId)
  const animal = animals.find(a => a.id === event?.animalId)

  const status = event ? getCareStatus(event.dueDate) : null
  const isComplete = !!event?.completedDate

  const onComplete = async () => {
    if (!event) return
    await bsCareService.completeCareEvent(event)
    navigation.goBack()
  }

  return { event, animal, status, isComplete, onComplete }
}
```

export function useCreateCareEventController(navigation, route) {
  const { animalId, templateId } = route.params
  const { animals } = useAnimalStore()
  const { careTemplates } = useAnimalTypeStore()
  const [selectedAnimalId, setSelectedAnimalId] = useState(animalId ?? '')
  const [name, setName] = useState('')
  const [type, setType] = useState<CareEventType>('careSingle')
  const [cycle, setCycle] = useState(0)
  const [dueDate, setDueDate] = useState(todayIso())
  const [contactName, setContactName] = useState('')
  const [contactPhone, setContactPhone] = useState('')
  const [notes, setNotes] = useState('')
  const [loading, setLoading] = useState(false)

  useEffect(() => {
    if (templateId) {
      const template = careTemplates.find(t => t.id === templateId)
      if (template) {
        setName(template.name)
        setType(template.type)
        setCycle(template.cycle)
        setContactName(template.contactName)
        setContactPhone(template.contactPhone)
      }
    }
  }, [templateId])

  const submit = async () => {
    if (!name.trim() || !selectedAnimalId) return
    setLoading(true)
    await bsCareService.createCareEvent({
      animalId: selectedAnimalId, name, type, cycle, dueDate,
      contactName, contactPhone, notes, templateId: templateId ?? '',
      completedDate: null, createdNextRecurringEvent: false,
      photoStorageRef: '', photoUrl: '',
    } as CareEvent)
    setLoading(false)
    navigation.goBack()
  }

  return { /* all state */ submit, animals, loading }
}
```

# Subscription Gating

- **Free tier**: 3 care events per animal. On create, check count for the target animal from the store.
- **Pro/Farm tier**: Unlimited care events.

# Notes

- Completing a recurring care event is the most important business logic in the app
- Always calculate next due date from `completedDate`, never from the original `dueDate`
- Template updates don't retroactively change existing care events
- The Cloud Function `onCareEventComplete` trigger is a safety net for the recurring event creation
- Care events from templates carry the `templateId` reference but are independent copies
- The cross-animal care view (Care tab) is the second-most-used screen after the Today Dashboard
- The `onSnapshot` real-time listener means completing a care event on one screen (e.g., Today Dashboard) instantly reflects on the Care tab and Animal Detail Care tab
- Controllers never import Firebase — they call `bsCareService` from Bootstrap and read from Zustand stores
- The store's `subscribe()` should be called once during app initialization (after auth)
