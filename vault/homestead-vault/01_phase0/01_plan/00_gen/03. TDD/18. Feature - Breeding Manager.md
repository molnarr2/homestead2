# Goal

Build the breeding manager with sire/dam pairing, auto-calculated gestation/incubation countdown with species-specific defaults and breed overrides, birth outcome recording with offspring auto-linking, and the breeding status state machine. This is the **#2 subscription conversion driver** — breeders obsess over due dates.

# Instructions

## Architecture Notes

1. `BreedingService` calls `@react-native-firebase/firestore` directly. All farm data lives under `/homestead/{homesteadId}/` subcollections. The service uses `this.homesteadRef` (a `DocumentReference` for `firestore().collection('homestead').doc(homesteadId)`) to access subcollections.
2. `admin.created_at` and `admin.updated_at` are `Tstamp`. Use `adminObject_default()` and `adminObject_updateLastUpdated()`.
3. `BreedingService.subscribe()` uses `this.homesteadRef.collection('breedingRecord').where(...).onSnapshot(...)` so the breeding list and dashboard countdown update in real-time.
4. `completeBirth` uses `firestore().batch()` to atomically update the breeding record + create offspring animal documents.
5. No `userId` field on breeding record or offspring documents. No `userId` in queries or method signatures. The homestead subcollection path provides scoping.
6. Service location: `apps/mobile/src/feature/breeding/`.
7. Bootstrap export: `bsBreedingService: IBreedingService` from `apps/mobile/src/Bootstrap.ts`.

## Breeding Service (`apps/mobile/src/feature/breeding/service/`)

### `IBreedingService.ts`
```typescript
import { IResult } from '../../../util/Result'
import { BreedingRecord, BirthOutcome } from '../../../schema/breeding/BreedingRecord'

export default interface IBreedingService {
  subscribe(onUpdate: (records: BreedingRecord[]) => void): () => void
  createBreedingRecord(record: BreedingRecord): Promise<IResult>
  completeBirth(recordId: string, outcome: BirthOutcome, dam: Animal): Promise<IResult>
  failBreeding(recordId: string): Promise<IResult>
}
```

### `BreedingService.ts`
```typescript
import firestore, { FirebaseFirestoreTypes } from '@react-native-firebase/firestore'
import { IResult, SuccessResult, ErrorResult } from '../../../util/Result'
import { BreedingRecord, BirthOutcome } from '../../../schema/breeding/BreedingRecord'
import { adminObject_default } from '../../../schema/object/AdminObject'
import Log from '../../../library/log/Log'

export default class BreedingService implements IBreedingService {
  private homesteadRef: FirebaseFirestoreTypes.DocumentReference

  subscribe(onUpdate: (records: BreedingRecord[]) => void): () => void {
    return this.homesteadRef
      .collection('breedingRecord')
      .where('admin.deleted', '==', false)
      .onSnapshot(
        snapshot => {
          const records: BreedingRecord[] = snapshot.docs.map(doc => ({
            ...doc.data(),
            id: doc.id,
          } as BreedingRecord))
          onUpdate(records)
        },
        error => {
          Log.error('BreedingService', 'subscribe error: ' + error.message)
        }
      )
  }

  async createBreedingRecord(record: BreedingRecord): Promise<IResult> {
    try {
      const docRef = this.homesteadRef.collection('breedingRecord').doc()
      record.id = docRef.id
      record.admin = adminObject_default()
      record.status = 'active'
      await docRef.set(record)
      return SuccessResult
    } catch (error: any) {
      Log.error('BreedingService', 'createBreedingRecord error: ' + error.message)
      return ErrorResult(error.message)
    }
  }

  async completeBirth(recordId: string, outcome: BirthOutcome, dam: Animal): Promise<IResult> {
    try {
      const batch = firestore().batch()
      const offspringIds: string[] = []

      // 1. Create offspring animal documents
      for (let i = 0; i < outcome.bornAlive; i++) {
        const offspringRef = this.homesteadRef.collection('animal').doc()
        offspringIds.push(offspringRef.id)
        batch.set(offspringRef, {
          id: offspringRef.id,
          name: `Baby ${i + 1}`,
          animalType: dam.animalType,
          animalTypeId: dam.animalTypeId,
          animalTypeLevel: dam.animalTypeLevel,
          breed: dam.breed,
          animalBreedId: dam.animalBreedId,
          birthday: outcome.birthDate,
          gender: 'unknown',
          color: '',
          register: '',
          state: 'own',
          notes: '',
          photoStorageRef: '',
          photoUrl: '',
          purchasePrice: 0,
          weight: 0,
          weightUnit: 'lbs',
          sireId: outcome.sireId ?? '',
          damId: dam.id,
          admin: adminObject_default(),
        })
      }

      // 2. Update the breeding record with birth outcome
      const recordRef = this.homesteadRef.collection('breedingRecord').doc(recordId)
      batch.update(recordRef, {
        status: 'completed',
        birthDate: outcome.birthDate,
        bornAlive: outcome.bornAlive,
        stillborn: outcome.stillborn,
        complications: outcome.complications,
        damCondition: outcome.damCondition,
        offspringIds,
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })

      // 3. Commit atomically
      await batch.commit()
      return SuccessResult
    } catch (error: any) {
      Log.error('BreedingService', 'completeBirth error: ' + error.message)
      return ErrorResult(error.message)
    }
  }

  async failBreeding(recordId: string): Promise<IResult> {
    try {
      await this.homesteadRef.collection('breedingRecord').doc(recordId).update({
        status: 'failed',
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })
      return SuccessResult
    } catch (error: any) {
      Log.error('BreedingService', 'failBreeding error: ' + error.message)
      return ErrorResult(error.message)
    }
  }
}
```

**Critical business logic — `completeBirth`:**
- Uses `firestore().batch()` directly for atomicity — offspring creation + breeding record update succeed or fail together.
- For each `bornAlive`, creates a new Animal document in `this.homesteadRef.collection('animal')` inheriting `animalType`, `breed`, `animalTypeId`, `animalTypeLevel`, `animalBreedId` from the dam. No `userId` on offspring documents.
- Offspring get `damId` and `sireId` linked, enabling lineage tracking.
- Offspring names default to "Baby 1", "Baby 2", etc. — user renames later.
- The `offspringIds` array on the breeding record provides the reverse link.
- All offspring appear in the Animals tab immediately (the AnimalStore's `onSnapshot` picks them up).

## BreedingStore (Zustand)

```typescript
import { create } from 'zustand'
import { BreedingRecord } from '../../schema/breeding/BreedingRecord'
import { bsBreedingService } from '../../Bootstrap'

interface BreedingStoreState {
  breedingRecords: BreedingRecord[]
  loading: boolean
  unsubscribe: (() => void) | null

  subscribe: () => void
  teardown: () => void
  clear: () => void
}

export const useBreedingStore = create<BreedingStoreState>((set, get) => ({
  breedingRecords: [],
  loading: true,
  unsubscribe: null,

  subscribe: () => {
    get().teardown()
    set({ loading: true })
    const unsub = bsBreedingService.subscribe((breedingRecords) => {
      set({ breedingRecords, loading: false })
    })
    set({ unsubscribe: unsub })
  },

  teardown: () => {
    const unsub = get().unsubscribe
    if (unsub) unsub()
    set({ unsubscribe: null })
  },

  clear: () => {
    get().teardown()
    set({ breedingRecords: [], loading: true, unsubscribe: null })
  },
}))
```

Uses `subscribe()` with `onSnapshot` — all breeding records (active, completed, failed) are streamed in real-time. The controller filters by status as needed.

## Create Breeding Record Screen

### `CreateBreedingRecordController.ts`
```typescript
export function useCreateBreedingRecordController(navigation, route) {
  const { animalId } = route.params  // dam
  const { animals } = useAnimalStore()
  const { getGestationDays } = useAnimalTypeStore()

  const dam = animals.find(a => a.id === animalId)

  const [sireId, setSireId] = useState('')
  const [sireName, setSireName] = useState('')
  const [breedingDate, setBreedingDate] = useState(todayIso())
  const [method, setMethod] = useState<BreedingMethod>('natural')
  const [notes, setNotes] = useState('')
  const [loading, setLoading] = useState(false)

  const gestationDays = dam ? getGestationDays(dam.animalTypeId, dam.animalBreedId) : 0
  const gestationStatus = breedingDate && gestationDays > 0
    ? calculateGestation(breedingDate, dam?.animalType ?? '', gestationDays)
    : null

  const onSireSelect = (id: string) => {
    setSireId(id)
    const sire = animals.find(a => a.id === id)
    setSireName(sire ? sire.name : '')
  }

  const submit = async () => {
    setLoading(true)
    await bsBreedingService.createBreedingRecord({
      animalId, sireId, sireName, breedingDate,
      expectedDueDate: gestationStatus?.expectedDueDate ?? '',
      method, notes, status: 'active',
    } as BreedingRecord)
    setLoading(false)
    navigation.goBack()
  }

  return {
    dam, sireId, sireName, onSireSelect, breedingDate, setBreedingDate,
    method, setMethod, notes, setNotes, gestationStatus, gestationDays,
    loading, submit, animals,
  }
}
```

export function useBreedingRecordDetailController(navigation, route) {
  const { recordId } = route.params
  const { breedingRecords } = useBreedingStore()
  const { animals } = useAnimalStore()
  const record = breedingRecords.find(r => r.id === recordId)
  const dam = animals.find(a => a.id === record?.animalId)
  const sire = record?.sireId ? animals.find(a => a.id === record.sireId) : null

  const gestationStatus = record && record.status === 'active'
    ? calculateGestation(record.breedingDate, dam?.animalType ?? '')
    : null

  const onRecordBirth = () => navigation.navigate('RecordBirthOutcome', { recordId })
  const onMarkFailed = async () => {
    await bsBreedingService.failBreeding(recordId)
    navigation.goBack()
  }

  const offspring = record?.offspringIds
    ? animals.filter(a => record.offspringIds.includes(a.id))
    : []

  return { record, dam, sire, gestationStatus, offspring, onRecordBirth, onMarkFailed }
}

export function useRecordBirthOutcomeController(navigation, route) {
  const { recordId } = route.params
  const { breedingRecords } = useBreedingStore()
  const { animals } = useAnimalStore()
  const record = breedingRecords.find(r => r.id === recordId)
  const dam = animals.find(a => a.id === record?.animalId)

  const [bornAlive, setBornAlive] = useState(1)
  const [stillborn, setStillborn] = useState(0)

      birthDate, bornAlive, stillborn, complications, damCondition,
      sireId: record.sireId,

  return {
    record, dam, birthDate, setBirthDate, bornAlive, setBornAlive,
    stillborn, setStillborn, complications, setComplications,
    damCondition, setDamCondition, loading, submit,
  }
}

# Notes

- The breeding state machine is simple: `active` → `completed` or `failed`. No transition back to `active`.
- Offspring auto-linking creates real Animal documents — they appear in the Animals tab immediately via the AnimalStore's `onSnapshot` listener.
- Sire can be from the user's herd (linked by ID) or an external sire (just a name string).
- The gestation countdown with progress bar creates daily engagement during breeding season — missing a due date means losing animals.
- Active breedings surface on the Today Dashboard for visibility.
- Progress bar caps at 100% if overdue (does not exceed).
- The `onSnapshot` real-time listener on breeding records means recording a birth instantly updates the breeding tab and dashboard countdown.
- Controllers never import Firebase — they call `bsBreedingService` from Bootstrap and read from Zustand stores.
- The store's `subscribe()` should be called once during app initialization (after auth). Takes no arguments — homesteadRef is already set on the service.
- The `completeBirth` batch write creates offspring animal documents that the AnimalStore's `onSnapshot` picks up automatically — no manual refresh needed.
- All data stored under `/homestead/{homesteadId}/breedingRecord/` and `/homestead/{homesteadId}/animal/` subcollections — no `userId` on documents.
