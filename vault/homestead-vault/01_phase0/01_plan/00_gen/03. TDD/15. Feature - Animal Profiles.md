# Goal

Build the Animals tab with animal list grouped by type, search/filter, animal CRUD, animal detail screen with all sub-sections (health records, breeding, care history, notes, weight logs), photo upload, and state management (own/sold/died/processed). The Animal Detail screen is the hub — all per-animal data is accessed from here.

# Instructions

## Architecture Notes

1. `AnimalService` calls `@react-native-firebase/firestore` and `@react-native-firebase/storage` directly.
2. `admin.created_at` and `admin.updated_at` are `Tstamp` (Firebase Timestamp | FieldValue). Use `adminObject_default()` and `adminObject_updateLastUpdated()` from `AdminObject.ts`.
3. `AnimalService.subscribe()` uses `this.homesteadRef.collection('animal').where(...).onSnapshot(...)` so the animal list updates in real-time without manual refetch.
4. Service location: `apps/mobile/src/feature/animal/service/` with interface + implementation.
5. Bootstrap export: `bsAnimalService: IAnimalService` exported from `apps/mobile/src/Bootstrap.ts`.

## Animal Service (`apps/mobile/src/feature/animal/service/`)

### `IAnimalService.ts`
```typescript
import { IResult } from '../../../util/Result'
import { Animal } from '../../../schema/animal/Animal'

export default interface IAnimalService {
  subscribe(onUpdate: (animals: Animal[]) => void): () => void
  createAnimal(animal: Animal, photoUri?: string): Promise<IResult>
  updateAnimal(animal: Animal, photoUri?: string): Promise<IResult>
  softDeleteAnimal(animalId: string): Promise<IResult>
  updateAnimalState(animalId: string, state: AnimalState): Promise<IResult>
}
```

### `AnimalService.ts`
```typescript
import firestore from '@react-native-firebase/firestore'
import storage from '@react-native-firebase/storage'
import { IResult, SuccessResult, ErrorResult } from '../../../util/Result'
import { Animal } from '../../../schema/animal/Animal'
import { adminObject_default, adminObject_updateLastUpdated } from '../../../schema/object/AdminObject'
import Log from '../../../library/log/Log'

export default class AnimalService implements IAnimalService {

  subscribe(onUpdate: (animals: Animal[]) => void): () => void {
    return this.homesteadRef
      .collection('animal')
      .where('admin.deleted', '==', false)
      .onSnapshot(
        snapshot => {
          const animals: Animal[] = snapshot.docs.map(doc => ({
            ...doc.data(),
            id: doc.id,
          } as Animal))
          onUpdate(animals)
        },
        error => {
          Log.error('AnimalService', 'subscribe error: ' + error.message)
        }
      )
  }

  async createAnimal(animal: Animal, photoUri?: string): Promise<IResult> {
    try {
      const docRef = this.homesteadRef.collection('animal').doc()
      animal.id = docRef.id
      animal.admin = adminObject_default()

      if (photoUri) {
        const storagePath = `homestead/${this.homesteadId}/animal/${animal.id}/photo.jpg`
        await storage().ref(storagePath).putFile(photoUri)
        const downloadUrl = await storage().ref(storagePath).getDownloadURL()
        animal.photoStorageRef = storagePath
        animal.photoUrl = downloadUrl
      }

      await docRef.set(animal)
      return SuccessResult
    } catch (error: any) {
      Log.error('AnimalService', 'createAnimal error: ' + error.message)
      return ErrorResult(error.message)
    }
  }

  async updateAnimal(animal: Animal, photoUri?: string): Promise<IResult> {
    try {
      animal.admin = adminObject_updateLastUpdated(animal.admin)

      if (photoUri) {
        const storagePath = `homestead/${this.homesteadId}/animal/${animal.id}/photo.jpg`
        await storage().ref(storagePath).putFile(photoUri)
        const downloadUrl = await storage().ref(storagePath).getDownloadURL()
        animal.photoStorageRef = storagePath
        animal.photoUrl = downloadUrl
      }

      await this.homesteadRef.collection('animal').doc(animal.id).set(animal)
      return SuccessResult
    } catch (error: any) {
      Log.error('AnimalService', 'updateAnimal error: ' + error.message)
      return ErrorResult(error.message)
    }
  }

  async softDeleteAnimal(animalId: string): Promise<IResult> {
    try {
      await this.homesteadRef.collection('animal').doc(animalId).update({
        'admin.deleted': true,
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })
      return SuccessResult
    } catch (error: any) {
      Log.error('AnimalService', 'softDeleteAnimal error: ' + error.message)
      return ErrorResult(error.message)
    }
  }

  async updateAnimalState(animalId: string, state: AnimalState): Promise<IResult> {
    try {
      await this.homesteadRef.collection('animal').doc(animalId).update({
        state,
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })
      return SuccessResult
    } catch (error: any) {
      Log.error('AnimalService', 'updateAnimalState error: ' + error.message)
      return ErrorResult(error.message)
    }
  }
}
```

**Key implementation details:**
- `subscribe()` returns an unsubscribe function from `onSnapshot`. The store calls this on teardown.
- Real-time listener means no `fetchAnimals()` — the store receives push updates whenever Firestore data changes (add, edit, delete by any client or Cloud Function).
- Photo upload uses `@react-native-firebase/storage` directly — no `ICloudPlatformStorage` wrapper.
- All Firestore calls use the homestead subcollection: `this.homesteadRef.collection('animal')...`.
- `adminObject_default()` sets `created_at` and `updated_at` to `serverTimestamp()`.
- Registered in `Bootstrap.ts` as `export const bsAnimalService: IAnimalService = new AnimalService()`.

## AnimalStore (Zustand)

```typescript
import { create } from 'zustand'
import { Animal } from '../../schema/animal/Animal'
import { bsAnimalService } from '../../Bootstrap'

interface AnimalStoreState {
  animals: Animal[]
  loading: boolean
  unsubscribe: (() => void) | null

  subscribe: () => void
  teardown: () => void
  clear: () => void
}

export const useAnimalStore = create<AnimalStoreState>((set, get) => ({
  animals: [],
  loading: true,
  unsubscribe: null,

  subscribe: () => {
    get().teardown()
    set({ loading: true })
    const unsub = bsAnimalService.subscribe((animals) => {
      set({ animals, loading: false })
    })
    set({ unsubscribe: unsub })
  },

  teardown: () => {
    const unsub = get().unsubscribe
    if (unsub) unsub()
    set({ unsubscribe: null })
  },

  clear: () => {
    get().teardown()
    set({ animals: [], loading: true, unsubscribe: null })
  },
}))
```

**Store notes:**
- `subscribe()` establishes a real-time listener.
- Store receives push updates via `onSnapshot` callback.
- `teardown()` cleans up the Firestore listener to prevent memory leaks.
- `clear()` calls `teardown()` and resets state (called on sign-out via `resetAllStores()`).

```typescript
export function useAnimalListController(navigation) {
  const { animals, loading } = useAnimalStore()
  const [searchQuery, setSearchQuery] = useState('')
  const [filterState, setFilterState] = useState<AnimalState | 'all'>('all')

  const filteredAnimals = animals.filter(a => {
    const matchesSearch = a.name.toLowerCase().includes(searchQuery.toLowerCase()) ||
      a.breed.toLowerCase().includes(searchQuery.toLowerCase())
    const matchesState = filterState === 'all' || a.state === filterState
    return matchesSearch && matchesState
  })

  const groupedAnimals = groupBy(filteredAnimals, 'animalType')

  const onAnimalPress = (animalId: string) => navigation.navigate('AnimalDetail', { animalId })
  const onCreateAnimal = () => navigation.navigate('CreateAnimal')

  return { groupedAnimals, searchQuery, setSearchQuery, filterState, setFilterState, loading, onAnimalPress, onCreateAnimal }
}
```

The store is already subscribed to real-time updates via `onSnapshot` — the controller simply reads from it, no `useFocusEffect` / `fetchAnimals()` call needed.

# Subscription Gating

- **Free tier**: Maximum 10 animals. On create, check count from the store. If at limit, show Pro paywall.
- **Pro/Farm tier**: Unlimited animals.

# Notes

- Animal Detail is the hub — the most important screen in the app
- All per-animal data (health, breeding, care, notes, weight) is accessed from tabs within Animal Detail
- State transitions: `own` → `sold` / `died` / `processed` (one-way, set via edit screen)
- Parent linking: `sireId` and `damId` reference other animal documents — enables lineage tracking
- Age calculated client-side from `birthday` using `calculateAnimalAge()` utility
- Photo upload uses `@react-native-firebase/storage` directly in the service — no intermediary
- Animals are scoped by homestead path (`/homestead/{homesteadId}/animal/`) — no userId field on animal documents
- Search is client-side (all animals loaded into Zustand store via onSnapshot). For very large herds (100+), consider pagination.
- The `onSnapshot` real-time listener means the animal list is always current — no pull-to-refresh needed, but can be kept for UX familiarity
- Controllers never import Firebase — they use `bsAnimalService` from Bootstrap and read from Zustand stores
- The store's `subscribe()` should be called once during app initialization (after auth), not per-screen focus
