# Goal

Build the authentication and onboarding feature screens. This is the first user-facing flow: Login, Register, Species Selection, and Loading. Follows the Controller + Screen separation pattern. Registration creates a user profile AND a homestead with the user as owner. After registration, users select which species they keep — this seeds starter animal types, breeds, and care templates into the homestead. Controllers use `bsAuthService` (Bootstrap singleton, thin wrapper around `FirebaseAuth`). User creation is handled by `UserService`, homestead creation by `HomesteadService`.

## 4. Login Screen

### `LoginController.ts`
```typescript
import { useState } from 'react'
import { bsAuthService } from '../../../../Bootstrap'

export function useLoginController(navigation: any) {
  const [email, setEmail] = useState('')
  const [password, setPassword] = useState('')
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState('')

  const login = async () => {
    setLoading(true)
    setError('')
    const result = await bsAuthService.signInWithEmailAndPassword(email, password)
    if (!result.success) {
      setError(result.error)
    }
    // On success: FirebaseAuth.onAuthStateChanged fires → authStore.isLoggedIn = true → RootNavigation switches to DrawerMain
    setLoading(false)
  }

  const goToRegister = () => navigation.navigate('Register')

  const forgotPassword = async () => {
    if (!email) {
      setError('Enter your email first')
      return
    }
    await bsAuthService.sendPasswordResetEmail(email)
  }

  return { email, setEmail, password, setPassword, loading, error, login, goToRegister, forgotPassword }
}
```

## 5. Register Screen

### `RegisterController.ts`
```typescript
import { useState } from 'react'
import { bsAuthService, bsUserService, bsHomesteadService } from '../../../../Bootstrap'
import { user_default } from '../../../../schema/user/User'

export function useRegisterController(navigation: any) {
  const [firstName, setFirstName] = useState('')
  const [lastName, setLastName] = useState('')
  const [email, setEmail] = useState('')
  const [password, setPassword] = useState('')
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState('')

  const register = async () => {
    if (!firstName.trim()) {
      setError('First name is required')
      return
    }
    if (!email.trim() || !email.includes('@')) {
      setError('Valid email is required')
      return
    }
    if (password.length < 6) {
      setError('Password must be at least 6 characters')
      return
    }

    setLoading(true)
    setError('')

    const result = await bsAuthService.createAccount(email, password)
    if (!result.success) {
      setError(result.error)
      setLoading(false)
      return
    }

    // Create user profile document in Firestore
    const userId = bsAuthService.currentUserId
    await bsUserService.createUser({ ...user_default(), id: userId, firstName, lastName, email })

    // Create the first homestead with user as owner
    const homesteadId = await bsHomesteadService.createHomestead('My Homestead', userId, `${firstName} ${lastName}`, email)
    // Store activeHomesteadId on user
    await bsUserService.setActiveHomestead(userId, homesteadId)

    // Navigate to species selection (onboarding)
    navigation.navigate('SpeciesSelection')
    setLoading(false)
  }

  return {
    firstName, setFirstName,
    lastName, setLastName,
    email, setEmail,
    password, setPassword,
    loading, error, register,
  }
}
```

### `RegisterScreen.tsx`

- Register button (shared `PrimaryButton`)
- "Already have an account? Login" link
- Error display
- Validation: email format, password minimum 6 chars, first name required

  { name: 'Alpaca', icon: 'llama' },
  { name: 'Llama', icon: 'llama' },
  { name: 'Donkey', icon: 'donkey' },
]

export function useSpeciesSelectionController(navigation: any) {
  const [selectedSpecies, setSelectedSpecies] = useState<string[]>([])
  const [loading, setLoading] = useState(false)

  const toggleSpecies = (species: string) => {
    setSelectedSpecies(prev =>
      prev.includes(species)
        ? prev.filter(s => s !== species)
        : [...prev, species]
    )
  }

  const complete = async () => {
    if (selectedSpecies.length === 0) return
    setLoading(true)

    // Save selected species to user document and seed starter playbooks
    // AnimalTypeService calls Firestore directly:
    //   firestore().collection('user').doc(userId).update({ selectedSpecies, onboardingComplete: true })
    //   + seeds animal types, breeds, care templates for each selected species
    await bsAnimalTypeService.seedStarterPlaybooks(homesteadId, selectedSpecies)

    setLoading(false)
    // Auth + onboarding complete → RootNavigation shows DrawerMain
  }

  const skip = async () => {
    // Mark onboarding complete without seeding
    // firestore().collection('user').doc(userId).update({ onboardingComplete: true })
    await bsAnimalTypeService.skipOnboarding()
  }

  return { availableSpecies: AVAILABLE_SPECIES, selectedSpecies, toggleSpecies, loading, complete, skip }
}

## 7. User and Homestead Services (replace AuthFeatureService)

`AuthFeatureService` no longer exists. Its responsibilities are split:
- **User creation** is handled by `UserService` (`apps/mobile/src/feature/user/service/UserService.ts`) via `bsUserService.createUser(...)`.
- **Homestead creation** is handled by `HomesteadService` (`apps/mobile/src/feature/homestead/service/HomesteadService.ts`) via `bsHomesteadService.createHomestead(...)`.

The registration flow in `RegisterController` calls both services sequentially (see Section 5).

Key conventions:
- `UserService.createUser(user)` writes to `firestore().collection('user').doc(userId).set(...)`
- `HomesteadService.createHomestead(name, userId, displayName, email)` writes to `firestore().collection('homestead').doc()` and creates a `member` subcollection entry with role `owner`
- Both use `adminObject_default()` which sets `Tstamp` server timestamps
- Error handling returns `IResult` (same pattern as `AuthService`)

## 8. AnimalTypeService — Seed Starter Playbooks

The `AnimalTypeService` (separate from auth, but called during onboarding) seeds Firestore directly. Takes `homesteadId` to write into the homestead's subcollection:

```typescript
// In apps/mobile/src/feature/animal-type/service/AnimalTypeService.ts
import firestore from '@react-native-firebase/firestore'

export default class AnimalTypeService {
  async seedStarterPlaybooks(homesteadId: string, selectedSpecies: string[]): Promise<void> {
    const batch = firestore().batch()
    const homesteadRef = firestore().collection('homestead').doc(homesteadId)

    for (const species of selectedSpecies) {
      const playbook = getStarterPlaybook(species) // returns breeds + care templates
      const typeRef = homesteadRef.collection('animalType').doc()
      batch.set(typeRef, {
        name: species,
        icon: playbook.icon,
        breeds: playbook.breeds,
        admin: adminObject_default(),
      })

      for (const template of playbook.careTemplates) {
        const templateRef = homesteadRef.collection('animalType').doc(typeRef.id).collection('careTemplate').doc()
        batch.set(templateRef, {
          ...template,
          animalTypeId: typeRef.id,
          admin: adminObject_default(),
        })
      }
    }

    // Mark onboarding complete on user doc via bsUserService
    const userId = bsAuthService.currentUserId
    batch.update(firestore().collection('user').doc(userId), {
      onboardingComplete: true,
    })

    await batch.commit()
  }

  async skipOnboarding(): Promise<void> {
    const userId = bsAuthService.currentUserId
    await firestore().collection('user').doc(userId).update({
      onboardingComplete: true,
    })
  }
}
```

# Notes

- Auth state changes automatically trigger navigation via authStore → RootNavigation. No manual navigation after login — FirebaseAuth `onAuthStateChanged` fires, `BehaviorSubject` emits, `authStore.isLoggedIn` updates, `RootNavigation` re-renders.
- Species selection happens once after registration — seeds starter data for the customization engine
- Starter playbooks include common breeds and care templates per species (e.g., Goat → Alpine, Nubian breeds + deworming schedule, hoof trimming)
- `FirebaseAuth` handles all auth error mapping (email-in-use, weak-password, invalid-email, etc.) — controllers just check `result.success`
- `FirebaseAuth` caches `isLoggedIn` in MMKV for instant app-launch state (no flash of login screen)
- `AuthFeatureService` no longer exists — user creation is in `UserService`, homestead creation is in `HomesteadService`. Both are accessed via Bootstrap singletons (`bsUserService`, `bsHomesteadService`).
- Keep the UI warm and welcoming — this is the first impression. Earth tones per theme colors.
- Validation: email format, password minimum 6 chars, first name required
- All Firestore writes use `adminObject_default()` which embeds server timestamps via `tstampServerTime()`
