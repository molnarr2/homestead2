## 2. Domain-Specific Status Color Tokens

The current `global.css` has generic status colors (`status-success`, `status-warning`, `status-error`, `status-info`). Domain-specific status tokens that map to care/health semantics may be added to `global.css`:

```css
/* Add to @theme block if not already mappable to existing tokens */
--color-status-overdue: #E53935;     /* same as status-error */
--color-status-due-today: #FF9800;   /* same as status-warning */
--color-status-upcoming: #4CAF50;    /* same as status-success */
--color-status-future: #9E9E9E;      /* neutral gray */
--color-status-active: #E53935;      /* withdrawal active = red */
--color-status-clear: #4CAF50;       /* withdrawal cleared = green */
```

Currently, the existing tokens can serve these purposes via aliasing:
- `status-error` → overdue / active withdrawal
- `status-warning` → due today
- `status-success` → upcoming / clear
- `border-dark` (#9E9E9E) → future

However, adding explicit semantic aliases improves readability in Tailwind classes: `bg-status-overdue` is clearer than `bg-status-error` in a care context.

### Status Colors in `colors.ts`

`colors.ts` uses generic status names. When the care/health features are built, consider adding domain-specific aliases:

```typescript
status: {
  success: '#4CAF50',
```

### Additional Border Tokens

Current `global.css` has `border-light`, `border-medium`, `border-dark`. If feature designs require additional border colors (e.g., colored borders for status indicators), they can be added to the `@theme` block as needed.

## 3. Design Philosophy

- **Color palette**: Warm earth tones (sage green, warm brown, golden amber) — conveys nature, homesteading, trust
- **Status colors**: Critical for this app — red (overdue/active withdrawal), amber (due today), green (clear/upcoming), gray (future)
- **Typography**: Clean system fonts, generous line height for readability in barn/outdoor conditions
- **Spacing**: Consistent 8px grid
- **Shadows**: Soft, minimal — card-based layout with gentle elevation
- **Corners**: Rounded (12-16px) for warmth
- **Touch targets**: Minimum 44px — important for use with dirty/gloved hands

## 4. Styling Conventions

- **Prefer Uniwind classes** in JSX for layout, colors, spacing: `className="bg-primary p-4 rounded-lg"`
- **Use `src/theme/` constants** for programmatic values only: tab bar tint, chart colors, dynamic calculations
- **No `react-native-paper`** — all components are custom or Uniwind-styled primitives
- **No `StyleSheet.create`** unless there's a measurable performance reason for a specific component

# Notes

- Uniwind uses the `className` prop on RN components (same API as NativeWind)
- Uniwind is a drop-in replacement for NativeWind with better RN 0.84+ compatibility
- No `tailwind.config.js` needed — TailwindCSS v4 uses CSS-based config exclusively
- No Uniwind babel plugin needed — just `module:@react-native/babel-preset`
- Status colors are business-critical: withdrawal calculator ACTIVE must be unmistakably red, CLEAR must be unmistakably green
- The theme is intentionally minimal — add tokens as features demand them, don't pre-create unused tokens
