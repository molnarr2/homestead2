# Goal

Build reusable UI components in `apps/mobile/src/components/` shared across multiple features. All styled with Uniwind (Tailwind CSS for React Native). These are the building blocks for feature screens. Designed for barn-friendly use — large touch targets, high contrast status indicators, clear visual hierarchy.

# References

- `apps/mobile/src/components/` — component library root
- `apps/mobile/src/feature/debug/DebugScreen.tsx` — showcases components
- `apps/mobile/src/feature/debug/DebugController.ts` — manages debug screen state
- `apps/mobile/src/theme/colors.ts` — color tokens for status colors

## 2. Component Specifications

### `ScreenContainer.tsx`
- Wraps `SafeAreaView` with consistent padding and background color
- Uniwind: `className="flex-1 bg-background"`
- Props: `children: ReactNode`, `className?: string`
- Optional scroll mode: `scrollable?: boolean` wraps children in `ScrollView`

### `SectionHeader.tsx`
- Section title with optional count badge and action button
- Props: `title: string`, `count?: number`, `action?: { label: string, onPress: () => void }`
- Uniwind: `className="flex-row items-center justify-between mb-2 mt-4"`
- Title: `className="text-lg font-bold text-text-primary"`
- Count rendered as a `Badge` component next to title

### `EmptyState.tsx`
- Centered message with icon for empty lists
- Props: `icon: string`, `title: string`, `subtitle?: string`
- Uniwind: `className="flex-1 items-center justify-center py-12"`
- Icon from `@react-native-vector-icons/material-design-icons`, size 48, text-secondary color
- Title: `className="text-lg font-semibold text-text-primary mt-4"`
- Subtitle: `className="text-sm text-text-secondary mt-1 text-center px-8"`

### `LoadingIndicator.tsx`
- Centered `ActivityIndicator` with optional message
- Props: `message?: string`, `size?: 'small' | 'large'`
- Uniwind: `className="flex-1 items-center justify-center"`

### `Avatar.tsx`
- Circular image with fallback initials (or animal icon for animal avatars)
- Sizes: `sm` (32), `md` (44), `lg` (64)
- Props: `uri?: string`, `name: string`, `size: 'sm' | 'md' | 'lg'`
- If `uri` present, show `Image` with rounded-full
- If no `uri`, show circle with initials (first letter of first + last name) on primary-light background

### `Badge.tsx`
- Small colored badge for counts and labels
- Props: `label: string`, `variant: 'primary' | 'secondary' | 'accent' | 'error' | 'warning' | 'success'`
- Uniwind: rounded-full, px-2, py-0.5, text-xs font-semibold
- Variant maps: primary → bg-primary text-white, error → bg-status-error text-white, etc.

### `StatusBadge.tsx`
- Color-coded status indicator for care/health statuses
- Variants with mapped colors:
  - `overdue` → red (bg-status-error)
  - `dueToday` → amber (bg-status-warning)
  - `upcoming` → green (bg-status-success)
  - `future` → gray (bg-border-medium)
  - `active` → red (bg-status-error)
  - `clear` → green (bg-status-success)
- Props: `status: 'overdue' | 'dueToday' | 'upcoming' | 'future' | 'active' | 'clear'`, `label?: string`
- Shows small colored dot + label text

### `Divider.tsx`
- Horizontal line separator
- Props: `className?: string`
- Default: `className="h-px bg-border-light my-2"`

### `ProgressBar.tsx`
- Horizontal progress bar (used for gestation countdown)
- Props: `progress: number` (0–100), `color?: string`, `label?: string`
- Track: `className="h-2 bg-border-light rounded-full"`
- Fill: dynamically set width as percentage, default color primary, rounded-full
- Optional label below: `className="text-xs text-text-secondary mt-1"`

### `PhotoPicker.tsx`
- Image display + tap to pick from camera/gallery
- Uses `react-native-image-picker` and `react-native-image-resizer`
- Props: `uri?: string`, `onPick: (uri: string) => void`, `size?: 'sm' | 'md' | 'lg'`
- Size maps: sm=64, md=96, lg=128
- If `uri` set, show image with rounded corners; if not, show placeholder icon
- On tap, show action sheet: "Take Photo" / "Choose from Library" / "Cancel"

### `AnimalCard.tsx`
- Displays: photo thumbnail, name, breed, age, state badge
- Optional: active withdrawal indicator (red dot), upcoming care indicator
- Props: `animal: Animal`, `onPress: () => void`, `showStatus?: boolean`
- Uniwind: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm flex-row"`
- Left: photo thumbnail or `Avatar` with animal icon (40×40)
- Center: name (bold), breed (secondary text), age (secondary text)
- Right: `StatusBadge` if active withdrawal or care due

### `CareEventCard.tsx`
- Displays: event name, animal name, due date, status color bar (left border)
- Status colors: overdue (red left border `border-l-4 border-status-error`), due-today (amber `border-status-warning`), upcoming (green `border-status-success`), future (gray `border-border-medium`)
- Action: "Mark Complete" button (small `SecondaryButton` or text link)
- Props: `event: CareEvent`, `animalName: string`, `onComplete: () => void`, `onPress: () => void`, `status: 'overdue' | 'dueToday' | 'upcoming' | 'future'`
- Uniwind: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm border-l-4"`

### `HealthRecordCard.tsx`
- Displays: record type icon, name, date, withdrawal status badge (if applicable)
- Props: `record: HealthRecord`, `onPress: () => void`
- Record type icons: vaccination → syringe, medication → pill, procedure → scissors, observation → eye, test → flask, deworming → bug
- Uniwind: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm flex-row"`

### `BreedingCard.tsx`
- Displays: dam name, sire name, breeding date, countdown/progress bar, status
- Active: shows days remaining + `ProgressBar`
- Completed: shows birth outcome summary
- Props: `record: BreedingRecord`, `damName: string`, `onPress: () => void`
- Uniwind: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm"`

### `ProductionCard.tsx`
- Displays: production type icon, quantity + unit, date, animal/group name
- Props: `log: ProductionLog`, `onPress: () => void`
- Icons: eggs → egg, milk → water, wool → content-cut, honey → beehive
- Uniwind: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm flex-row"`

### `NoteCard.tsx`
- Displays: text snippet (2-line max), tags as colored chips, date, photo thumbnail if present
- Props: `note: Note`, `onPress: () => void`
- Uniwind: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm"`
- Tags shown as row of `TagChip` components

### `DashboardAlertCard.tsx`
- Displays: alert type icon, title, subtitle, color-coded urgency
- Variants: withdrawal active (red), breeding due soon (amber), overdue care (red)
- Props: `type: 'withdrawal' | 'breeding' | 'care'`, `title: string`, `subtitle: string`, `urgency: 'high' | 'medium' | 'low'`, `onPress: () => void`
- Urgency maps: high → border-status-error bg-red-50, medium → border-status-warning bg-amber-50, low → border-border-medium bg-surface
- Uniwind: `className="rounded-xl p-4 mb-3 border-l-4 flex-row"`

### `DatePicker.tsx`
- Wraps `@react-native-community/datetimepicker`
- Shows formatted date in a tappable field, opens native picker on press
- Props: `label: string`, `value: Date`, `onChange: (date: Date) => void`, `mode?: 'date' | 'datetime'`
- Display: formatted with `date-fns` `format()`, styled like TextInput (border, rounded, padding)
- On Android: shows picker in dialog mode; on iOS: inline or modal

### `NumberInput.tsx`
- Numeric input with +/- stepper buttons on sides
- Props: `label: string`, `value: number`, `onChange: (value: number) => void`, `min?: number`, `max?: number`, `step?: number`, `unit?: string`
- Layout: `[-]` button | value display | `[+]` button
- Stepper buttons use `IconButton` or similar touchable
- Respects min/max bounds, step defaults to 1

### `DropdownSelect.tsx`
- Tappable field that opens `SelectDialog` with options
- Props: `label: string`, `value: string`, `options: { label: string, value: string }[]`, `onChange: (value: string) => void`, `placeholder?: string`
- Display: styled like TextInput with chevron-down icon on right
- On press: opens `SelectDialog` (already built) with the options list

### `TagChip.tsx`
- Small colored rounded tag
- Color mapped to tag name:
  - Health → red (status-error)
  - Behavior → blue (status-info)
  - Breeding → purple (#9C27B0)
  - Feed → orange (status-warning)
  - Production → green (status-success)
  - General → gray (border-dark)
- Props: `tag: string`, `onPress?: () => void`, `selected?: boolean`
- Uniwind: `className="rounded-full px-3 py-1 mr-2 mb-1"`

### `TagSelector.tsx`
- Row of `TagChip` components with multi-select capability
- Props: `tags: string[]`, `selected: string[]`, `onToggle: (tag: string) => void`
- Layout: `className="flex-row flex-wrap"`
- Selected tags have solid background; unselected have outline style

## 4. Uniwind Styling Conventions

- Use `className` prop on all RN components
- Avoid inline `style` objects unless dynamic values are needed (e.g., dynamic width for ProgressBar fill)
- Common patterns:
  - Cards: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm"`
  - Text primary: `className="text-base text-text-primary"`
  - Text secondary: `className="text-sm text-text-secondary"`
  - Dividers: `className="h-px bg-border-light my-2"`
  - Status borders: `className="border-l-4 border-status-error"` (for overdue care)
  - Touch targets: minimum height 48px (`min-h-[48px]`)
  - Section spacing: `mt-4 mb-2` on headers, `mb-3` on cards

# Notes

- All components are functional components with TypeScript props interfaces
- No `react-native-paper` — everything is built from RN primitives + Uniwind
- Components are presentational only (no state management or service calls) — pass callbacks via props
- Minimum touch target size: 48px height — barn-friendly design
- Status colors must be consistent across all components: red=urgent/active, amber=warning/today, green=healthy/clear, gray=neutral/future
- Button components follow a shared pattern for loading/disabled handling; inputs follow a shared label+error pattern
- Cards accept domain schema types as props (Animal, CareEvent, HealthRecord, etc.) — import from `apps/mobile/src/schema/`
- Icons use `@react-native-vector-icons/material-design-icons` throughout
- `DropdownSelect` composes with `SelectDialog` — don't build a new selection UI
- `DatePicker` uses native pickers — no custom date UI needed
