# Goal

Define the Firestore data model (collections/subcollections), security rules, storage rules, indexes, and common query patterns. All farm data is nested under `/homestead/{homesteadId}/`. User profiles and feedback remain at root level. Access is controlled by homestead membership roles. Services reference Firestore collections directly by name. AdminObject fields (`updated_at`, `created_at`) are Firebase Timestamps (`Tstamp`).

**Important:** We share one Firebase project with the existing production app. The `user` collection is reused (with a `schemaVersion` field to distinguish v2 users). Homestead subcollections use singular names without a `_v2` suffix — the homestead path provides a separate namespace that prevents collision with existing production data.

Note: No `userId` field on any homestead subcollection document. No `_v2` suffix — homestead subcollections are in a separate namespace.

**All documents include `admin: { deleted: boolean, updated_at: Tstamp, created_at: Tstamp }`.**

The `admin` field uses Firebase Timestamps:
- On create: `updated_at = serverTimestamp()`, `created_at = serverTimestamp()`
- On update: `updated_at = serverTimestamp()` (created_at unchanged)
- Via `adminObject_default()` and `adminObject_updateLastUpdated()` from `AdminObject.ts`

## 2. Direct Collection References

Services reference collections via the homestead document reference:

```typescript
import firestore from '@react-native-firebase/firestore'

// Homestead reference (homesteadId comes from store/context)
const homesteadRef = firestore().collection('homestead').doc(homesteadId)

// In AnimalService:
homesteadRef.collection('animal')

// In CareService:
homesteadRef.collection('careEvent')

// In AnimalTypeService (subcollection):
homesteadRef.collection('animalType').doc(typeId).collection('breed')
```

Collection name strings are used exactly once per service. If a collection name needs to change, you change it in one service file.

## 3. Firestore Security Rules (`firebase-rule/firestore.rules`)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isAuth() {
      return request.auth != null;
    }

    function isOwner(userId) {
      return request.auth.uid == userId;
    }

    // Check if the user is a member of the homestead (has any role)
    function isHomesteadMember(homesteadId) {
      return isAuth() && exists(/databases/$(database)/documents/homestead/$(homesteadId)/member/$(request.auth.uid));
    }

    // User profile — owner only
    match /user/{userId} {
      allow read: if isAuth() && isOwner(userId);
      allow create: if isAuth() && isOwner(userId);
      allow update: if isAuth() && isOwner(userId);
      allow delete: if false;

      match /device/{deviceId} {
        allow read, write: if isAuth() && isOwner(userId);
      }
    }

    // Homestead — members only
    match /homestead/{homesteadId} {
      allow read: if isHomesteadMember(homesteadId);
      allow create: if isAuth();
      allow update: if isHomesteadMember(homesteadId);
      allow delete: if false;

      // Members — members can read, only homestead members can manage
      match /member/{memberId} {
        allow read: if isHomesteadMember(homesteadId);
        allow create: if isHomesteadMember(homesteadId);
        allow update: if isHomesteadMember(homesteadId);
        allow delete: if false;
      }

      // All farm data — homestead members only
      match /animal/{animalId} {
        allow read: if isHomesteadMember(homesteadId);
        allow create: if isHomesteadMember(homesteadId);
        allow update: if isHomesteadMember(homesteadId);
        allow delete: if false;
      }

      match /careEvent/{eventId} {
        allow read: if isHomesteadMember(homesteadId);

        allow update: if isHomesteadMember(homesteadId);

      match /healthRecord/{recordId} {

        allow delete: if false;

      match /breedingRecord/{recordId} {

        allow read: if isHomesteadMember(homesteadId);

        allow update: if isHomesteadMember(homesteadId);
        allow delete: if false;
      }

      match /weightLog/{logId} {
        allow read: if isHomesteadMember(homesteadId);
        allow create: if isHomesteadMember(homesteadId);
        allow update: if isHomesteadMember(homesteadId);
        allow delete: if false;
      }

      match /productionLog/{logId} {
        allow read: if isHomesteadMember(homesteadId);
        allow create: if isHomesteadMember(homesteadId);
        allow update: if isHomesteadMember(homesteadId);

        allow update: if isHomesteadMember(homesteadId);
```

## 4. Storage Rules (`firebase-rule/storage.rules`)

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    // User-specific storage (avatars)
    match /user/{userId}/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }
    // Homestead storage (animal photos, note photos, etc.)
    match /homestead/{homesteadId}/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }
  }
}
```

## 6. Common Query Patterns (Service Code)

Queries are written inside feature services using `@react-native-firebase/firestore`:

```typescript
import firestore from '@react-native-firebase/firestore'

// Get homestead reference
const homesteadRef = firestore().collection('homestead').doc(homesteadId)

// --- AnimalService: Real-time animal list ---
homesteadRef
  .collection('animal')
  .where('admin.deleted', '==', false)
  .onSnapshot(snapshot => {
    const animals = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() } as Animal))
    callback(animals)
  })

// --- CareService: Real-time incomplete care events ---
homesteadRef
  .collection('careEvent')
  .where('admin.deleted', '==', false)
  .where('completedDate', '==', null)
  .orderBy('dueDate')
  .onSnapshot(snapshot => {
    const events = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() } as CareEvent))
    callback(events)
  })

// --- HealthService: Active withdrawals (fetch) ---
const snapshot = await homesteadRef
  .collection('healthRecord')
  .where('admin.deleted', '==', false)
  .where('withdrawalPeriodDays', '>', 0)
  .get()

// --- BreedingService: Active breedings (real-time) ---
homesteadRef
  .collection('breedingRecord')
  .where('admin.deleted', '==', false)
  .where('status', '==', 'active')
  .orderBy('expectedDueDate')
  .onSnapshot(...)

// --- Create a document ---
await homesteadRef.collection('animal').add({
  ...animalData,
  admin: adminObject_default(),
})

// --- Soft delete ---
await homesteadRef.collection('animal').doc(animalId).update({
  'admin.deleted': true,
  'admin.updated_at': tstampServerTime(),
})
```

# Notes

- **Homestead is the central entity.** All farm data lives under `/homestead/{homesteadId}/`. Access is controlled by homestead membership — any user with a role in the homestead's `member` subcollection can read/write farm data.
- No `userId` field on farm data documents — ownership is implicit by the homestead path.
- `user` collection is shared with the existing production app (via `schemaVersion` field).
- No `_v2` suffix on homestead subcollections — different namespace prevents collision.
- No hard deletes — all documents use soft-delete via `admin.deleted`.
- Security rules check `isHomesteadMember()` which does an `exists()` call on the member subcollection.
- Subcollection rules under `animalType` also check homestead membership (not just `isAuth()`).
- `CareEvent.dueDate` and `CareEvent.completedDate` are `Tstamp` (Firebase Timestamps). `completedDate` is `null` when not completed (not empty string). This enables native Firestore range queries.
- `admin.updated_at` and `admin.created_at` are `Tstamp` (Firebase `Timestamp` | `FieldValue`). Use `tstampServerTime()` for writes, `tstampToDate()` for reads when you need a JS `Date`.
- When reading documents from Firestore snapshots, timestamps come back as `FirebaseFirestoreTypes.Timestamp` objects. The `Tstamp.ts` utilities handle conversion.
