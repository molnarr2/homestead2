Create the minimal set of Firebase Cloud Functions needed for MVP. Functions handle server-side logic that cannot be trusted to the client: care reminder notifications, breeding due notifications, withdrawal clearance notifications, and recurring event safety nets. Lives in `/functions`.

## 2. Notification Functions

### `onCareEventDue.ts`
- Scheduled function (runs daily at 7:00 AM UTC)
- Uses collection group query: `db.collectionGroup('careEvent')` to query across all homesteads
- Queries where `completedDate == null` AND `dueDate <= today`
- Groups by homesteadId (extracted from document path), looks up homestead members, sends push notification to all members: "You have {count} care items due today"
- Creates notification data for in-app display

### `onBreedingDue.ts`
- Scheduled function (runs daily)
- Uses collection group query: `db.collectionGroup('breedingRecord')` to query across all homesteads
- Queries active breedingRecords where `expectedDueDate` is within 3 days
- Gets homesteadId from document path, looks up members, sends push notification to their devices: "{animalName} is due in {days} days"

### `onWithdrawalClear.ts`
- Scheduled function (runs daily)
- Uses collection group query: `db.collectionGroup('healthRecord')` to query across all homesteads
- Queries healthRecords with `withdrawalPeriodDays > 0`
- Checks if withdrawal period ended today (compare `date` + `withdrawalPeriodDays` against today)
- Gets homesteadId from document path, looks up members, sends notification to their devices: "Withdrawal period cleared for {animalName}"

## 3. Trigger Functions

### `onCareEventComplete.ts`
- Firestore trigger: `onUpdate` on `/homestead/{homesteadId}/careEvent/{eventId}`
- When `completedDate` changes from null to a Timestamp AND `type == 'careRecurring'`:
  1. Verify `createdNextRecurringEvent` is false
  2. Create new careEvent with `dueDate = completedDate + cycle` (both are Timestamps)
  3. New event gets `admin: { deleted: false, updated_at: Timestamp.now(), created_at: Timestamp.now() }`
  4. Set `createdNextRecurringEvent = true` on the completed event
- This is a safety net — the client also creates the next event to avoid latency

```typescript
import { onDocumentUpdated } from 'firebase-functions/v2/firestore'
import { getFirestore, Timestamp } from 'firebase-admin/firestore'

export const onCareEventComplete = onDocumentUpdated(
  'homestead/{homesteadId}/careEvent/{eventId}',
  async (event) => {
    const before = event.data?.before.data()
    const after = event.data?.after.data()
    if (!before || !after) return

    const wasIncomplete = before.completedDate == null
    const isNowComplete = after.completedDate != null
    const isRecurring = after.type === 'careRecurring'
    const alreadyCreatedNext = after.createdNextRecurringEvent === true

    if (wasIncomplete && isNowComplete && isRecurring && !alreadyCreatedNext) {
      const db = getFirestore()
      const homesteadId = event.params.homesteadId

      const newDueDate = addDays(after.completedDate as Timestamp, after.cycle)

      const homesteadRef = db.collection('homestead').doc(homesteadId)
      await homesteadRef.collection('careEvent').add({
        ...after,
        id: '',  // auto-generated
        dueDate: newDueDate,
        completedDate: null,
        createdNextRecurringEvent: false,
        admin: {
          deleted: false,
          updated_at: Timestamp.now(),
          created_at: Timestamp.now(),
        },
      })

      await event.data?.after.ref.update({
        createdNextRecurringEvent: true,
        'admin.updated_at': Timestamp.now(),
      })
    }
  }
)
```

## 4. Scheduled Functions

### `dailyCareReminder.ts`
- Runs daily at 7:00 AM UTC
- Uses collection group query: `db.collectionGroup('careEvent')` to aggregate across all homesteads
- Groups by homesteadId (extracted from document path: `homestead/{homesteadId}/careEvent/{eventId}`)
- For each homestead, looks up members via `homestead/{homesteadId}/member` subcollection, then gets their device tokens from `/user/{userId}/device/{deviceId}.tokenId`
- Sends push notifications via FCM to all members of affected homesteads

```typescript
import { onSchedule } from 'firebase-functions/v2/scheduler'
import { getFirestore, Timestamp } from 'firebase-admin/firestore'
import { getMessaging } from 'firebase-admin/messaging'

export const dailyCareReminder = onSchedule('every day 07:00', async () => {
  const db = getFirestore()
  const today = new Date()
  today.setHours(23, 59, 59, 999)

  const snapshot = await db.collectionGroup('careEvent')
    .where('admin.deleted', '==', false)
    .where('completedDate', '==', null)
    .where('dueDate', '<=', Timestamp.fromDate(today))
    .get()

  // Group by homesteadId (extracted from document path)
  const byHomestead: Record<string, number> = {}
  snapshot.docs.forEach(doc => {
    // Path: homestead/{homesteadId}/careEvent/{eventId}
    const homesteadId = doc.ref.parent.parent?.id
    if (homesteadId) {
      byHomestead[homesteadId] = (byHomestead[homesteadId] || 0) + 1
    }
  })

  for (const [homesteadId, count] of Object.entries(byHomestead)) {
    // Look up all members of this homestead
    const membersSnap = await db.collection('homestead').doc(homesteadId).collection('member').get()
    const memberUserIds = membersSnap.docs.map(d => d.id)

    // Get device tokens for all members
    const tokens: string[] = []
    for (const userId of memberUserIds) {
      const devicesSnap = await db.collection('user').doc(userId).collection('device').get()
      devicesSnap.docs.forEach(d => {
        const token = d.data().tokenId
        if (token) tokens.push(token)
      })
    }

    if (tokens.length > 0) {
      await getMessaging().sendEachForMulticast({
        tokens,
        notification: {
          title: 'Care Reminder',
          body: `You have ${count} care item${count > 1 ? 's' : ''} due today`,
        },
      })
    }
  }
})
```

## 6. Timestamp Handling in Functions

`admin` fields are Firestore Timestamps. `CareEvent.dueDate` and `CareEvent.completedDate` are also Timestamps, enabling native Firestore range queries. `completedDate == null` replaces `completedDate === ''` for incomplete events. Cloud Functions code uses `firebase-admin` Timestamp utilities:

```typescript
import { Timestamp } from 'firebase-admin/firestore'

// Create a new admin object
const admin = {
  deleted: false,
  updated_at: Timestamp.now(),
  created_at: Timestamp.now(),
}

// Compare dates — dueDate and completedDate are already Timestamps
const dueDate = doc.data().dueDate as Timestamp
const today = Timestamp.now()
if (dueDate.toMillis() <= today.toMillis()) {
  // event is due or overdue
}

// completedDate: null means incomplete, Timestamp means completed
const completedDate = doc.data().completedDate  // null | Timestamp
if (completedDate == null) {
  // event is not yet completed
}

// Add days to a timestamp
function addDays(timestamp: Timestamp, days: number): Timestamp {
  const date = timestamp.toDate()
  date.setDate(date.getDate() + days)
  return Timestamp.fromDate(date)
}
```

# Notes

- Keep functions minimal for MVP — only build what's absolutely needed
- Notifications require the user's FCM token stored in `/user/{userId}/device/{deviceId}`. Cloud Functions find affected users by extracting `homesteadId` from the document path, then looking up members via the `homestead/{homesteadId}/member` subcollection.
- Use `firebase-admin` for server-side Firestore operations (not the client SDK)
- All functions should include error handling and logging
- Do NOT build: payments processing (RevenueCat handles this), analytics aggregation, AI features
- The `onCareEventComplete` trigger is a safety net — the client also creates the next recurring event to avoid latency
- Scheduled functions use Firebase's built-in scheduler (no external cron needed)
- Timestamps in Cloud Functions come from `firebase-admin/firestore` (`Timestamp.now()`, `Timestamp.fromDate()`) — these are the server-side equivalents of the client's `serverTimestamp()` and `Timestamp.fromDate()`
