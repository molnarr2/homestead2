# Goal

Build the Profile screen and Settings screen accessible from the side menu. Includes user info display, edit profile with avatar upload, app settings (notification preferences, unit preferences), send feedback form, and logout with confirmation.

**Architecture notes:**
- `IProfileService.ts` (interface) + `ProfileService.ts` (implementation) — Interface + Implementation pattern
- `ProfileService` handles only: avatar upload and feedback submission. User CRUD is in `UserService`.
- `ProfileService` calls `@react-native-firebase/firestore` and `@react-native-firebase/storage` directly
- User docs at ROOT level: `firestore().collection('user').doc(userId)` (singular, not under homestead)
- Feedback at ROOT level: `firestore().collection('feedback')` (singular, no `_v2`)
- Settings stored via MMKV directly using `createMMKV()` from `react-native-mmkv`
- Logout uses `bsAuthService` which wraps `FirebaseAuth` at `apps/mobile/src/library/cloudplatform/firebase/FirebaseAuth.ts`
- AdminObject uses Firebase Timestamps (`Tstamp`)
- Service exported from `Bootstrap.ts` as `bsProfileService`

## Profile Screen

### `ProfileController.ts`
```typescript
import { bsAuthService } from '../../../Bootstrap'
import { useUserStore } from '../../../store/useUserStore'
import { useAnimalStore } from '../../../store/useAnimalStore'

export function useProfileController(navigation: any) {
  const { user } = useUserStore()
  const { animals } = useAnimalStore()

  const animalCount = animals.filter(a => a.state === 'own').length
  const animalTypeCount = [...new Set(animals.map(a => a.animalType))].length

  const onEditProfile = () => navigation.navigate('EditProfile')
  const onSettings = () => navigation.navigate('Settings')
  const onSubscription = () => navigation.navigate('Subscription')
  const onSendFeedback = () => navigation.navigate('SendFeedback')

    bsAuthService.signOut()
    resetAllStores()
  }

  return {
    user, animalCount, animalTypeCount,
    onEditProfile, onSettings, onSubscription, onSendFeedback, onLogout,
  }
}
```

## Edit Profile Screen

### `EditProfileController.ts`
```typescript
import { useState } from 'react'
import { bsProfileService, bsUserService } from '../../../Bootstrap'
import { useUserStore } from '../../../store/useUserStore'

export function useEditProfileController(navigation: any) {
  const { user, setUser } = useUserStore()

  const [firstName, setFirstName] = useState(user?.firstName ?? '')
  const [lastName, setLastName] = useState(user?.lastName ?? '')
  const [email] = useState(user?.email ?? '')
  const [photoUri, setPhotoUri] = useState('')
  const [loading, setLoading] = useState(false)

  const save = async () => {
    if (!user) return
    setLoading(true)

    // User field updates (firstName, lastName) handled by UserService
    await bsUserService.updateUser(user.id, { firstName, lastName })

    // Avatar upload handled by ProfileService
    if (photoUri) {
      await bsProfileService.updateAvatar(user.id, photoUri)
    }

    setUser({ ...user, firstName, lastName })
    setLoading(false)
    navigation.goBack()
  }

  return {
    firstName, setFirstName, lastName, setLastName, email,
    photoUri, setPhotoUri, user, loading, save,
  }
}
```

### `EditProfileScreen.tsx`
- Avatar (large, tappable with `PhotoPicker` to change)
- First Name input
- Last Name input
- Email (display only, grayed out)
- Save button

## Settings Screen

### `SettingsController.ts`

Settings are stored locally in MMKV using `createMMKV()` directly — no `ICacheService` wrapper. MMKV is imported and used inline.

```typescript
import { useState, useEffect } from 'react'
import { createMMKV } from 'react-native-mmkv'

const settingsStorage = createMMKV()

const SETTINGS_KEYS = {
  notificationsEnabled: 'settings-notifications-enabled',
  weightUnit: 'settings-weight-unit',
  milkUnit: 'settings-milk-unit',
}

export function useSettingsController() {
  const [notificationsEnabled, setNotificationsEnabled] = useState(true)
  const [weightUnit, setWeightUnit] = useState<'lbs' | 'kg'>('lbs')
  const [milkUnit, setMilkUnit] = useState<'gallons' | 'liters'>('gallons')

  useEffect(() => {
    const storedNotif = settingsStorage.getBoolean(SETTINGS_KEYS.notificationsEnabled)
    if (storedNotif !== undefined) setNotificationsEnabled(storedNotif)

    const storedWeight = settingsStorage.getString(SETTINGS_KEYS.weightUnit)
    if (storedWeight === 'lbs' || storedWeight === 'kg') setWeightUnit(storedWeight)

    const storedMilk = settingsStorage.getString(SETTINGS_KEYS.milkUnit)
    if (storedMilk === 'gallons' || storedMilk === 'liters') setMilkUnit(storedMilk)
  }, [])

  const onToggleNotifications = (value: boolean) => {
    setNotificationsEnabled(value)
    settingsStorage.set(SETTINGS_KEYS.notificationsEnabled, value)
  }

  const onChangeWeightUnit = (value: 'lbs' | 'kg') => {
    setWeightUnit(value)
    settingsStorage.set(SETTINGS_KEYS.weightUnit, value)
  }

  const onChangeMilkUnit = (value: 'gallons' | 'liters') => {
    setMilkUnit(value)
    settingsStorage.set(SETTINGS_KEYS.milkUnit, value)
  }

  return {
    notificationsEnabled, onToggleNotifications,
    weightUnit, onChangeWeightUnit,
    milkUnit, onChangeMilkUnit,
  }
}
```

### `SettingsScreen.tsx`
- **Notifications section**:
  - Enable/disable push notifications (Switch toggle)
- **Units section**:
  - Weight: lbs / kg (selector)
  - Milk volume: gallons / liters (selector)
- **App Info section**:
  - App version
  - Build number

## Send Feedback Screen

### `SendFeedbackController.ts`
```typescript
import { useState } from 'react'
import { Platform } from 'react-native'
import { getVersion } from 'react-native-device-info'
import { bsProfileService } from '../../../Bootstrap'
import { useUserStore } from '../../../store/useUserStore'

export function useSendFeedbackController(navigation: any) {
  const { user } = useUserStore()
  const [rating, setRating] = useState(0)
  const [feedback, setFeedback] = useState('')
  const [loading, setLoading] = useState(false)

  const submit = async () => {
    if (!feedback.trim()) return
    setLoading(true)
    await bsProfileService.submitFeedback({
      id: '',
      admin: {} as any,
      userId: user?.id ?? '',
      email: user?.email ?? '',
      rating,
      feedback,
      os: Platform.OS === 'ios' ? 'iOS' : 'Android',
      version: getVersion(),
    })
    setLoading(false)
    navigation.goBack()
  }

  return { rating, setRating, feedback, setFeedback, loading, submit }
}
```

### `SendFeedbackScreen.tsx`
- Rating selector: 1-5 stars
- Feedback text (multiline TextInput)
- Submit button
- Thank you message on success

## Profile Service

### Interface (`apps/mobile/src/feature/profile/service/IProfileService.ts`)

ProfileService only handles avatar upload and feedback submission. User CRUD is in `UserService`.

```typescript
import { IResult } from '../../../util/Result'
import { Feedback } from '../../../schema/profile/Feedback'

export interface IProfileService {
  updateAvatar(userId: string, photoUri: string): Promise<IResult>
  submitFeedback(feedback: Feedback): Promise<IResult>
}
```

### Implementation (`apps/mobile/src/feature/profile/service/ProfileService.ts`)

The service calls `@react-native-firebase/firestore` and `@react-native-firebase/storage` directly. Types are imported from schema — not re-defined here (D9).

```typescript
import firestore from '@react-native-firebase/firestore'
import storage from '@react-native-firebase/storage'
import { IResult, SuccessResult, ErrorResult } from '../../../util/Result'
import { adminObject_default } from '../../../schema/object/AdminObject'
import Log from '../../../library/log/Log'
import { Feedback } from '../../../schema/profile/Feedback'
import { IProfileService } from './IProfileService'

export default class ProfileService implements IProfileService {

  async updateAvatar(
    userId: string,
    photoUri: string,
  ): Promise<IResult> {
    try {
      const storagePath = `user/${userId}/avatar/${Date.now()}.jpg`
      await storage().ref(storagePath).putFile(photoUri)
      const downloadUrl = await storage().ref(storagePath).getDownloadURL()

      await firestore().collection('user').doc(userId).update({
        photoStorageRef: storagePath,
        photoUrl: downloadUrl,
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })
      return SuccessResult
    } catch (error) {
      Log.error('ProfileService', 'updateAvatar failed: ' + error)
      return ErrorResult('Failed to update avatar: ' + error)
    }
  }

  async submitFeedback(feedback: Feedback): Promise<IResult> {
    try {
      const docRef = firestore().collection('feedback').doc()
      await docRef.set({
        ...feedback,
        id: docRef.id,
        admin: adminObject_default(),
      })
      return SuccessResult
    } catch (error) {
      Log.error('ProfileService', 'submitFeedback failed: ' + error)
      return ErrorResult('Failed to submit feedback: ' + error)
    }
  }
}
```

# Notes

- Profile is accessed from the side menu, not a bottom tab
- Email is read-only on the edit screen (changing email requires Firebase Auth re-authentication which is not MVP)
- Settings are stored locally in MMKV via `createMMKV()` directly — not synced to Firestore, device-specific preferences
- No `ICacheService` wrapper for settings — the controller creates its own MMKV instance and reads/writes keys directly
- Feedback is write-only from the client perspective — no read access in security rules
- Feedback collection at root level: `firestore().collection('feedback')` (singular, no `_v2`)
- User collection at root level: `firestore().collection('user')` (singular, not under homestead)
- ProfileService only handles: avatar upload and feedback submission. User CRUD is in `UserService`.
- Logout calls `bsAuthService.signOut()` which delegates to the existing `FirebaseAuth.signOut()` → `auth().signOut()`
- Logout also calls `resetAllStores()` to clear all Zustand stores
- Photo avatar stored at `user/{userId}/avatar/{filename}` in Firebase Storage (user-scoped, not homestead)
- AdminObject timestamps use Firebase `Tstamp` (server timestamps)
