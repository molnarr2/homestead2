# Goal

Create all local scripts in `/scripts` for running, cleaning, deploying, and managing the Homestead project. Also includes Firebase deploy scripts for rules and functions. Scripts reflect the monorepo structure with `apps/mobile/` and `firebase.json` at the project root.

# References

- `firebase-rule/` â€” Rules to deploy
- `functions/` â€” Functions to deploy

# Instructions

## 1. Scripts

### `scripts/run-ios.sh`
```bash
#!/bin/bash
set -e
echo "Running Homestead on iOS..."
SIMULATOR="${1:-iPhone 16}"
cd "$(dirname "$0")/../apps/mobile"
npx react-native run-ios --simulator "$SIMULATOR"
```

### `scripts/run-android.sh`
```bash
#!/bin/bash
set -e
echo "Running Homestead on Android..."
cd "$(dirname "$0")/../apps/mobile"

if [ -z "$JAVA_HOME" ]; then
  echo "Error: JAVA_HOME is not set"
  exit 1
fi
if [ -z "$ANDROID_HOME" ]; then
  echo "Error: ANDROID_HOME is not set"
  exit 1
fi

npx react-native run-android
```

### `scripts/clean-ios.sh`
```bash
#!/bin/bash
set -e
echo "Cleaning iOS build..."
cd "$(dirname "$0")/../apps/mobile"
cd ios
rm -rf build Pods Podfile.lock
pod cache clean --all
cd ..
rm -rf node_modules
npm install
cd ios
pod install
echo "iOS clean complete."
```

### `scripts/clean-android.sh`
```bash
#!/bin/bash
set -e
echo "Cleaning Android build..."
cd "$(dirname "$0")/../apps/mobile"
cd android
./gradlew clean
cd ..
rm -rf node_modules
npm install
echo "Android clean complete."
```

### `scripts/deploy-rules.sh`
```bash
#!/bin/bash
set -e
echo "Deploying Firestore and Storage rules..."
cd "$(dirname "$0")/.."
firebase deploy --only firestore:rules,storage --project <PROJECT_ID>
echo "Rules deployed."
```

### `scripts/deploy-functions.sh`
```bash
#!/bin/bash
set -e
echo "Building Cloud Functions..."
cd "$(dirname "$0")/../functions"
npm run build
echo "Deploying Cloud Functions..."
cd "$(dirname "$0")/.."
firebase deploy --only functions --project <PROJECT_ID>
echo "Functions deployed."
```

### `scripts/deploy-all.sh`
```bash
#!/bin/bash
set -e
echo "Deploying everything..."
"$(dirname "$0")/deploy-rules.sh"
"$(dirname "$0")/deploy-functions.sh"
echo "All deployments complete."
```

### `scripts/test.sh`
```bash
#!/bin/bash
set -e
echo "Running tests..."
cd "$(dirname "$0")/../apps/mobile"
npm test
echo "Tests complete."
```

## 2. Script Permissions

All scripts must be executable:
```bash
chmod +x scripts/*.sh
```

## 3. Firebase Project Setup

Create `firebase.json` **at the project root** (`/homestead2/firebase.json`):
```json
{
  "firestore": {
    "rules": "firebase-rule/firestore.rules",
    "indexes": "firebase-rule/firestore.indexes.json"
  },
  "storage": {
    "rules": "firebase-rule/storage.rules"
  },
  "functions": {
    "source": "functions"
  }
}
```

## 5. Bootstrap.ts Final Shape

By the time all features are built, `apps/mobile/src/Bootstrap.ts` will export:

```typescript
import FirebaseAuth from './library/cloudplatform/firebase/FirebaseAuth'
import RevenueCatInAppPurchases from './library/purchases/revenuecat/RevenueCatInAppPurchases'
import NoteService from './feature/notes/service/NoteService'
import ProfileService from './feature/profile/service/ProfileService'
import AnimalTypeService from './feature/customization/service/AnimalTypeService'
import SubscriptionService from './feature/subscription/service/SubscriptionService'
// ... other feature services

const firebaseAuth = new FirebaseAuth()
const revenueCat = new RevenueCatInAppPurchases()

const noteService = new NoteService()
const profileService = new ProfileService()
const animalTypeService = new AnimalTypeService()
const subscriptionService = new SubscriptionService(revenueCat)
// ... other feature services

export const bsFirebaseAuth = firebaseAuth
export const bsNoteService = noteService
export const bsProfileService = profileService
export const bsAnimalTypeService = animalTypeService
export const bsSubscriptionService = subscriptionService
// ... other feature service exports
```

| Export | Type | Added In |
|--------|------|----------|
| `bsFirebaseAuth` | `FirebaseAuth` | Foundation |
| `bsAnimalService` | `AnimalService` | TDD - Animal Profiles |
| `bsCareService` | `CareService` | TDD - Care Reminders |
| `bsHealthService` | `HealthService` | TDD - Health Records |
| `bsBreedingService` | `BreedingService` | TDD - Breeding Manager |
| `bsProductionService` | `ProductionService` | TDD - Production Tracking |
| `bsNoteService` | `NoteService` | TDD 20 - Notes & Observations |
| `bsProfileService` | `ProfileService` | TDD 21 - Profile & Settings |
| `bsAnimalTypeService` | `AnimalTypeService` | TDD 22 - Customization Engine |
| `bsSubscriptionService` | `SubscriptionService` | TDD 23 - Subscription & Paywall |

Each feature TDD specifies "Bootstrap Registration" â€” this means adding the instantiation and export at that step.

# Notes

- Replace `<PROJECT_ID>` with actual Firebase project ID once created
- Scripts use `set -e` to fail fast on errors
- All scripts use relative paths from the `scripts/` directory via `$(dirname "$0")`
- Scripts now reference `apps/mobile/` instead of `HomesteadApp/` for the monorepo structure
- `firebase.json` remains at the project root â€” Firebase CLI runs from the repo root
- `functions/` and `firebase-rule/` also remain at the project root
- No CI/CD for MVP â€” manual deploys via scripts
- Consider adding a `scripts/setup.sh` for first-time developer setup (install deps, pods, configure Firebase)
- No plugin interfaces in Bootstrap â€” services are instantiated directly with their concrete dependencies
- Services call `@react-native-firebase/*` directly â€” no intermediate `ICloudPlatform` wrappers
- The only library-level classes that persist are `FirebaseAuth` (auth operations) and `RevenueCatInAppPurchases` (in-app purchases) â€” these are thin wrappers around their respective SDKs and are injected into feature services via Bootstrap
