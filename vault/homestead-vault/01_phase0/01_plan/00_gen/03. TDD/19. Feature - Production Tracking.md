# Goal

Build the Production tab with daily logging for eggs, milk, fiber, honey, meat, and other production types. Supports per-animal or per-group (flock/herd) logging with daily/weekly/monthly aggregation and trend visibility. Production logging is a daily ritual — the tab must support one-tap quick entry.

# Instructions

## Architecture Notes

1. `ProductionService` calls `@react-native-firebase/firestore` directly. All farm data lives under `/homestead/{homesteadId}/` subcollections. The service uses `this.homesteadRef` (a `DocumentReference` for `firestore().collection('homestead').doc(homesteadId)`) to access subcollections.
2. `admin.created_at` and `admin.updated_at` are `Tstamp`. Use `adminObject_default()` and `adminObject_updateLastUpdated()`.
3. Production logs are historical data fetched on demand — no `onSnapshot` listener, use standard `get()` queries.
4. No `userId` field on production log documents. No `userId` in queries or method signatures. The homestead subcollection path provides scoping.
5. Service location: `apps/mobile/src/feature/production/`.
6. Bootstrap export: `bsProductionService: IProductionService` from `apps/mobile/src/Bootstrap.ts`.

## Production Service (`apps/mobile/src/feature/production/service/`)

### `IProductionService.ts`
```typescript
import { IResult } from '../../../util/Result'
import { ProductionLog } from '../../../schema/production/ProductionLog'

export default interface IProductionService {
  fetchProductionLogs(): Promise<ProductionLog[]>
  fetchProductionLogsByAnimal(animalId: string): Promise<ProductionLog[]>
  createProductionLog(log: ProductionLog): Promise<IResult>
}
```

### `ProductionService.ts`
```typescript
import firestore, { FirebaseFirestoreTypes } from '@react-native-firebase/firestore'
import { IResult, SuccessResult, ErrorResult } from '../../../util/Result'
import { ProductionLog } from '../../../schema/production/ProductionLog'
import { adminObject_default } from '../../../schema/object/AdminObject'
import Log from '../../../library/log/Log'

export default class ProductionService implements IProductionService {
  private homesteadRef: FirebaseFirestoreTypes.DocumentReference

  async fetchProductionLogs(): Promise<ProductionLog[]> {
    try {
      const snapshot = await this.homesteadRef
        .collection('productionLog')
        .where('admin.deleted', '==', false)
        .orderBy('date', 'desc')
        .get()

      return snapshot.docs.map(doc => ({
        ...doc.data(),
        id: doc.id,
      } as ProductionLog))
    } catch (error: any) {
      Log.error('ProductionService', 'fetchProductionLogs error: ' + error.message)
      return []
    }
  }

  async fetchProductionLogsByAnimal(animalId: string): Promise<ProductionLog[]> {
    try {
      const snapshot = await this.homesteadRef
        .collection('productionLog')
        .where('animalId', '==', animalId)
        .where('admin.deleted', '==', false)
        .orderBy('date', 'desc')
        .get()

      return snapshot.docs.map(doc => ({
        ...doc.data(),
        id: doc.id,
      } as ProductionLog))
    } catch (error: any) {
      Log.error('ProductionService', 'fetchProductionLogsByAnimal error: ' + error.message)
      return []
    }
  }

  async createProductionLog(log: ProductionLog): Promise<IResult> {
    try {
      const docRef = this.homesteadRef.collection('productionLog').doc()
      log.id = docRef.id
      log.admin = adminObject_default()
      await docRef.set(log)
      return SuccessResult
    } catch (error: any) {
      Log.error('ProductionService', 'createProductionLog error: ' + error.message)
      return ErrorResult(error.message)
    }
  }
}
```

**Key implementation details:**
- All queries use `this.homesteadRef.collection('productionLog')` — the singular `productionLog` subcollection under `/homestead/{homesteadId}/`.
- No `userId` field on documents, no `userId` in queries. Scoping is provided by the homestead subcollection path.
- No `onSnapshot` — production logs are historical data. Fetched on demand when viewing the Production tab.
- Simple CRUD with no cross-service side effects.
- `adminObject_default()` handles Firebase Timestamp initialization.

## ProductionStore (Zustand)

```typescript
import { create } from 'zustand'
import { ProductionLog } from '../../schema/production/ProductionLog'
import { bsProductionService } from '../../Bootstrap'

interface ProductionStoreState {
  productionLogs: ProductionLog[]
  loading: boolean

  fetchProductionLogs: () => Promise<void>
  fetchProductionLogsByAnimal: (animalId: string) => Promise<void>
  clear: () => void
}

export const useProductionStore = create<ProductionStoreState>((set) => ({
  productionLogs: [],
  loading: false,

  fetchProductionLogs: async () => {
    set({ loading: true })
    const logs = await bsProductionService.fetchProductionLogs()
    set({ productionLogs: logs, loading: false })
  },

  fetchProductionLogsByAnimal: async (animalId) => {
    set({ loading: true })
    const logs = await bsProductionService.fetchProductionLogsByAnimal(animalId)
    set({ productionLogs: logs, loading: false })
  },

  clear: () => set({ productionLogs: [], loading: false }),
}))
```

**Key difference from animal/care/breeding stores:** No `subscribe()` — production logs use fetch-on-demand since they are historical data. Uses `useFocusEffect` in the controller to refetch on screen focus.

## Production List Screen (Production Tab)

### `ProductionListController.ts`
```typescript
export function useProductionListController(navigation) {
  const { productionLogs, fetchProductionLogs, loading } = useProductionStore()
  const [selectedType, setSelectedType] = useState<ProductionType | 'all'>('all')
  const [refreshing, setRefreshing] = useState(false)
  const today = todayIso()

  useFocusEffect(useCallback(() => {
    fetchProductionLogs()
  }, []))

  const filteredLogs = selectedType === 'all'
    ? productionLogs
    : productionLogs.filter(l => l.productionType === selectedType)

  const dailySummary = aggregateProduction(filteredLogs, today)

  const recentLogs = filteredLogs
    .filter(l => daysBetween(l.date, today) <= 7)
    .sort((a, b) => b.date.localeCompare(a.date))

  const onCreateLog = (type?: ProductionType) =>
    navigation.navigate('CreateProductionLog', { type })

  const onRefresh = async () => {
    setRefreshing(true)
    await fetchProductionLogs()
    setRefreshing(false)
  }

  return {
    dailySummary, recentLogs, selectedType, setSelectedType,
    loading, refreshing, onRefresh, onCreateLog,
  }
}
```

**Key difference from animal/care controllers:** Uses `useFocusEffect` to refetch on screen focus since there's no real-time listener. Pull-to-refresh is implemented for manual refreshing.

## Create Production Log Screen

### `CreateProductionLogController.ts`
```typescript
export function useCreateProductionLogController(navigation, route) {
  const { animalId: initialAnimalId, groupName: initialGroup, type: initialType } = route.params ?? {}
  const { animals } = useAnimalStore()

  const [productionType, setProductionType] = useState<ProductionType>(initialType ?? 'eggs')
  const [quantity, setQuantity] = useState(0)
  const [unit, setUnit] = useState(getDefaultUnit(initialType ?? 'eggs'))
  const [date, setDate] = useState(todayIso())
  const [animalId, setAnimalId] = useState(initialAnimalId ?? '')
  const [groupName, setGroupName] = useState(initialGroup ?? '')
  const [notes, setNotes] = useState('')
  const [logMode, setLogMode] = useState<'animal' | 'group'>(initialAnimalId ? 'animal' : 'group')
  const [loading, setLoading] = useState(false)

  useEffect(() => {
    setUnit(getDefaultUnit(productionType))
  }, [productionType])

  const submit = async () => {
    if (quantity <= 0) return
    setLoading(true)
    await bsProductionService.createProductionLog({
      productionType, quantity, unit, date, notes,
      animalId: logMode === 'animal' ? animalId : '',
      groupName: logMode === 'group' ? groupName : '',
    } as ProductionLog)
    setLoading(false)
    navigation.goBack()
  }

  return {
    productionType, setProductionType, quantity, setQuantity, unit, setUnit,
    date, setDate, animalId, setAnimalId, groupName, setGroupName,
    notes, setNotes, logMode, setLogMode, animals, loading, submit,
  }
}
```

# Notes

- Production logging is a daily ritual — the quick-log entry makes it as fast as possible
- Per-animal OR per-group, never both — `animalId` and `groupName` are mutually exclusive
- Aggregation is client-side — all logs loaded into Zustand store, filtered and summed in the controller
- Trend visibility is important for farmers tracking seasonal patterns
- For MVP, a simple bar chart is sufficient — no need for a heavy charting library
- Units are flexible — while defaults are provided, users can log milk in liters instead of gallons
- No `onSnapshot` needed — production logs are historical data fetched on demand
- Controllers never import Firebase — they call `bsProductionService` from Bootstrap and read from Zustand stores
- Unlike animal/care/breeding stores, the production store uses explicit `fetch` actions with `useFocusEffect` for screen-focus refresh
- All data stored under `/homestead/{homesteadId}/productionLog/` subcollection — no `userId` on documents
