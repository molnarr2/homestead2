# Goal

Build the Home tab (Today Dashboard) — the primary screen users see after login. This is the single screen that drives daily app opens. It aggregates overdue care, due-today care, active withdrawal alerts, active breeding countdowns, and recent activity into one actionable view. Data is real-time where it makes sense — care events and active breedings update automatically via onSnapshot listeners in their respective services.

## 2. Real-Time Data Flow

The Today Dashboard is powered by Zustand stores that are backed by services using onSnapshot. The data flow:

```
Firestore onSnapshot → Service callback → Zustand store set() → Component re-render
```

### Real-time (via onSnapshot):
- **Care events** — `CareService.subscribe()` → `useCareStore` — overdue and due-today lists update automatically when care events are completed or new ones become due
- **Active breedings** — `BreedingService.subscribeToActiveBreedings()` → `useBreedingStore` — breeding countdown updates when records are modified

### Computed client-side (from store data):
- **Active withdrawals** — computed from `useHealthStore.healthRecords` using `WithdrawalUtility.getActiveWithdrawals()`. Health records are fetched on focus (not onSnapshot — less frequent changes).

### Fetched on focus:
- **Recent activity** — `ActivityService.fetchRecentActivity()` or aggregated from multiple stores. Fetched via `useFocusEffect`, not real-time.

## 3. HomeController.ts

```typescript
import { useState, useCallback } from 'react'
import { useFocusEffect } from '@react-navigation/native'
import { useUserStore } from '../../store/userStore'
import { useCareStore } from '../../store/careStore'
import { useBreedingStore } from '../../store/breedingStore'
import { useHealthStore } from '../../store/healthStore'
import { getActiveWithdrawals, WithdrawalResult } from '../../util/WithdrawalUtility'
import { getCareStatus } from '../../util/CareUtility'

export function useHomeController(navigation: any) {
  const user = useUserStore(s => s.user)
  const careEvents = useCareStore(s => s.careEvents)
  const breedingRecords = useBreedingStore(s => s.breedingRecords)
  const healthRecords = useHealthStore(s => s.healthRecords)
  const fetchHealthRecords = useHealthStore(s => s.fetchHealthRecords)
  const [refreshing, setRefreshing] = useState(false)

  // Health records fetched on focus (not real-time)
  useFocusEffect(
    useCallback(() => {
      fetchHealthRecords()
    }, [])
  )

  // Care events are real-time via onSnapshot — just derive overdue/dueToday from store
  const overdueEvents = careEvents.filter(e => getCareStatus(e) === 'overdue')
  const dueTodayEvents = careEvents.filter(e => getCareStatus(e) === 'dueToday')

  // Active breedings from real-time store
  const activeBreedings = breedingRecords.filter(r => r.status === 'active')

  // Withdrawals computed client-side from health records
  const activeWithdrawals: WithdrawalResult[] = getActiveWithdrawals(healthRecords)

  // Pull-to-refresh: re-fetch health records (care + breeding are already real-time)
  const onRefresh = async () => {
    setRefreshing(true)
    await fetchHealthRecords()
    setRefreshing(false)
  }

  // Navigation actions
  const onCareEventPress = (eventId: string) =>
    navigation.navigate('CareEventDetail', { eventId })
  const onAnimalPress = (animalId: string) =>
    navigation.navigate('AnimalDetail', { animalId })
  const onBreedingPress = (recordId: string) =>
    navigation.navigate('BreedingRecordDetail', { recordId })

  // Greeting based on time of day
  const getGreeting = (): string => {
    const hour = new Date().getHours()
    if (hour < 12) return 'Good morning'
    if (hour < 17) return 'Good afternoon'
    return 'Good evening'
  }

  return {
    user,
    greeting: getGreeting(),
    overdueEvents,
    dueTodayEvents,
    activeWithdrawals,
    activeBreedings,
    refreshing,
    onRefresh,
    onCareEventPress,
    onAnimalPress,
    onBreedingPress,
  }
}
```

## 6. CareService — OnSnapshot Pattern

The care events driving the dashboard come from a real-time subscription:

```typescript
// apps/mobile/src/feature/care/service/CareService.ts
import { dateToTstamp } from '../../../schema/type/Tstamp'
import CareEvent from '../../../schema/care/CareEvent'

export default class CareService implements ICareService {

  subscribe(onUpdate: (events: CareEvent[]) => void): () => void {
    return this.homesteadRef
      .collection('careEvent')
      .where('admin.deleted', '==', false)
      .where('completedDate', '==', null)
      .onSnapshot(snapshot => {
        const events = snapshot.docs.map(doc => ({
          id: doc.id,
          ...doc.data(),
        } as CareEvent))
        onUpdate(events)
      })
  }

  async completeCareEvent(eventId: string): Promise<void> {
    await this.homesteadRef
      .collection('careEvent')
      .doc(eventId)
      .update({
        completedDate: dateToTstamp(new Date()),
        'admin.updated_at': firestore.FieldValue.serverTimestamp(),
      })
    // onSnapshot will automatically push the updated list (this event drops out of incomplete query)
  }
}
```

### CareStore integration:

```typescript
// apps/mobile/src/store/careStore.ts
import { create } from 'zustand'
import { bsCareService } from '../Bootstrap'
import CareEvent from '../schema/care/CareEvent'

interface CareState {
  careEvents: CareEvent[]
  loading: boolean
  unsubscribe: (() => void) | null

  subscribe: () => void
  teardown: () => void
  clear: () => void
}

export const useCareStore = create<CareState>((set, get) => ({
  careEvents: [],
  loading: true,
  unsubscribe: null,

  subscribe: () => {
    get().teardown()
    set({ loading: true })
    const unsub = bsCareService.subscribe((careEvents) => {
      set({ careEvents, loading: false })
    })
    set({ unsubscribe: unsub })
  },

  teardown: () => {
    const unsub = get().unsubscribe
    if (unsub) unsub()
    set({ unsubscribe: null })
  },

  clear: () => {
    get().teardown()
    set({ careEvents: [], loading: true, unsubscribe: null })
  },
}))
```

The store's `subscribe()` is called once when the user logs in (e.g., from `MainScreen` mount or Bootstrap initialization). After that, every Firestore change automatically flows through to the dashboard — no manual refresh needed for care data.

## 7. CareUtility — Status Classification

```typescript
// apps/mobile/src/util/CareUtility.ts
import { tstampToDate } from '../schema/type/Tstamp'
import { startOfDay, differenceInDays } from 'date-fns'
import CareEvent from '../schema/care/CareEvent'

export type CareStatus = 'overdue' | 'dueToday' | 'upcoming' | 'future'

export function getCareStatus(event: CareEvent): CareStatus {
  const today = startOfDay(new Date())
  const dueDate = startOfDay(tstampToDate(event.dueDate) ?? new Date())
  const diff = differenceInDays(dueDate, today)

  if (diff < 0) return 'overdue'
  if (diff === 0) return 'dueToday'
  if (diff <= 3) return 'upcoming'
  return 'future'
}

export function getDaysOverdue(event: CareEvent): number {
  const today = startOfDay(new Date())
  const dueDate = startOfDay(tstampToDate(event.dueDate) ?? new Date())
  return Math.abs(differenceInDays(dueDate, today))
}
```

## 8. WithdrawalUtility — Active Withdrawal Computation

```typescript
// apps/mobile/src/util/WithdrawalUtility.ts
import { tstampToDate } from '../schema/type/Tstamp'
import { addDays, differenceInDays, isAfter } from 'date-fns'
import HealthRecord from '../schema/health/HealthRecord'

export interface WithdrawalResult {
  animalId: string
  animalName: string
  medicationName: string
  withdrawalType: 'meat' | 'milk' | 'eggs'
  daysRemaining: number
  clearDate: Date
}

export function getActiveWithdrawals(healthRecords: HealthRecord[]): WithdrawalResult[] {
  const today = new Date()
  const results: WithdrawalResult[] = []

  for (const record of healthRecords) {
    if (!record.withdrawalPeriodDays || record.withdrawalPeriodDays <= 0) continue

    const recordDate = tstampToDate(record.date)
    if (!recordDate) continue

    const clearDate = addDays(recordDate, record.withdrawalPeriodDays)

    if (isAfter(clearDate, today)) {
      results.push({
        animalId: record.animalId,
        animalName: record.animalName ?? '',
        medicationName: record.medicationName ?? record.name,
        withdrawalType: record.withdrawalType ?? 'meat',
        daysRemaining: differenceInDays(clearDate, today),
        clearDate,
      })
    }
  }

  return results.sort((a, b) => a.daysRemaining - b.daysRemaining)
}
```

# Notes

- The Today Dashboard is read-only — it surfaces data from other domains. No create/edit actions on this screen.
- Queries are cross-animal: care events, health records, breeding records all queried within homestead subcollections (not per-animal)
- Active withdrawals are computed client-side from health records (`getActiveWithdrawals()`) — no server-side computation needed
- Care events and active breedings are real-time via onSnapshot — completing a care event on the Care tab immediately removes it from the dashboard's overdue/due-today lists
- Health records are fetched on focus (not onSnapshot) — withdrawal data is less frequently changing
- The dashboard should feel like a morning farm checklist — "here's what needs attention today"
- If all sections are empty, the "All caught up!" empty state makes the app feel rewarding
- Performance: care event query filters `completedDate == null` and `admin.deleted == false` server-side — only incomplete events reach the client
- Pull-to-refresh only re-fetches health records (for withdrawal recalculation) — care and breeding data are already live
- Greeting is time-of-day aware: morning (before 12), afternoon (12-17), evening (after 17)
- No FAB on this screen — the dashboard is for viewing, not creating
