# Goal

Create the `Bootstrap.ts` composition root that instantiates and exports all services. Bootstrap wires services with interfaces. AuthService is at `core/service/auth/`, a thin wrapper around FirebaseAuth. UserService handles user document CRUD. Each feature service is added as that feature is implemented. Services call `@react-native-firebase/*` directly — there are no plugin interfaces.

# References

- `apps/mobile/src/library/cloudplatform/firebase/FirebaseAuth.ts` — uses `@react-native-firebase/auth` directly, rxjs `BehaviorSubject` for `loggedIn`, and MMKV for persistence
- `apps/mobile/src/util/Result.ts` — `IResult`, `SuccessResult`, `ErrorResult`
- `apps/mobile/src/library/log/Log.ts` — Singleton logger with Crashlytics

# Instructions

## 1. Auth Service (`core/service/auth/`)

AuthService is a thin wrapper around FirebaseAuth — only auth operations, no user document CRUD. It lives at `core/service/auth/` with separate interface and implementation files.

### `IAuthService.ts`

```typescript
import { IResult } from '../../../util/Result'
import { Subscribable } from 'rxjs'

export default interface IAuthService {
  readonly currentUserId: string
  readonly isLoggedIn: boolean
  readonly isAnonymous: boolean
  readonly loggedIn: Subscribable<boolean>

  createAccount(email: string, password: string): Promise<IResult>
  createAccountAnonymously(): Promise<IResult>
  signin(email: string, password: string): Promise<IResult>
  signout(): void
  sendPasswordResetEmail(email: string): Promise<void>
  deleteAuthAccount(): Promise<void>
  reauthenticate(email: string, password: string): Promise<IResult>
  linkUsernamePassword(email: string, password: string): Promise<IResult>
}
```

### `AuthService.ts`

```typescript
import { IFirebaseAuth } from '../../../library/cloudplatform/firebase/FirebaseAuth'
import type IAuthService from './IAuthService'
import { IResult } from '../../../util/Result'
import { Subscribable } from 'rxjs'

export default class AuthService implements IAuthService {
  private auth: IFirebaseAuth

  constructor(auth: IFirebaseAuth) {
    this.auth = auth
  }

  get currentUserId(): string { return this.auth.currentUserId }
  get isLoggedIn(): boolean { return this.auth.isLoggedIn }
  get isAnonymous(): boolean { return this.auth.isAnonymous }
  get loggedIn(): Subscribable<boolean> { return this.auth.loggedIn }

  createAccount(email: string, password: string): Promise<IResult> {
    return this.auth.createUserWithEmailAndPassword(email, password)
  }

  createAccountAnonymously(): Promise<IResult> {
    return this.auth.createAccountAnonymously()
  }

  signin(email: string, password: string): Promise<IResult> {
    return this.auth.signInWithEmailAndPassword(email, password)
  }

  signout(): void { this.auth.signOut() }

  sendPasswordResetEmail(email: string): Promise<void> {
    return this.auth.sendPasswordResetEmail(email)
  }

  deleteAuthAccount(): Promise<void> {
    return this.auth.deleteAuthAccount()
  }

  reauthenticate(email: string, password: string): Promise<IResult> {
    return this.auth.reauthenticate(email, password)
  }

  linkUsernamePassword(email: string, password: string): Promise<IResult> {
    return this.auth.linkUsernamePassword(email, password)
  }
}
```

## 2. Feature Service Pattern

Each feature service (Animal, Care, Health, etc.) follows this pattern. Every service has an **Interface + Implementation pair**: `I{Domain}Service.ts` + `{Domain}Service.ts`.

Services take no constructor dependencies (minimal injection). They get `homesteadId` from `useHomesteadStore` and access Firestore subcollections under the homestead document.

```typescript
// IAnimalService.ts
import { IResult } from '../../../util/Result'

export default interface IAnimalService {
  subscribe(callback: (animals: Animal[]) => void): () => void
  create(animal: Partial<Animal>): Promise<IResult>
  update(id: string, updates: Partial<Animal>): Promise<IResult>
  softDelete(id: string): Promise<IResult>
}
```

```typescript
// AnimalService.ts
import firestore from '@react-native-firebase/firestore'
import { useHomesteadStore } from '../../../store/homesteadStore'
import type IAnimalService from './IAnimalService'
import Log from '../../../library/log/Log'

export default class AnimalService implements IAnimalService {
  private get homesteadId(): string {
    return useHomesteadStore.getState().homesteadId
  }

  private get collection() {
    return firestore().collection('homestead').doc(this.homesteadId).collection('animal')
  }

  subscribe(callback: (animals: Animal[]) => void): () => void {
    return this.collection
      .where('admin.deleted', '==', false)
      .onSnapshot(snapshot => {
        const animals = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() } as Animal))
        callback(animals)
      }, error => {
        Log.error('AnimalService', 'subscribe error: ' + error)
      })
  }

  async create(animal: Partial<Animal>): Promise<IResult> { /* ... */ }
  async update(id: string, updates: Partial<Animal>): Promise<IResult> { /* ... */ }
  async softDelete(id: string): Promise<IResult> { /* ... */ }
}
```

Key rules:
- Services call `@react-native-firebase/firestore` directly
- Services access collections via: `firestore().collection('homestead').doc(homesteadId).collection('animal')`
- Services get `homesteadId` from `useHomesteadStore.getState().homesteadId` — no constructor deps needed
- Every service has `I{Domain}Service.ts` (interface) + `{Domain}Service.ts` (implementation)
- `subscribe()` returns an unsubscribe function for real-time listeners
- All Firebase imports stay inside service files — never leak to controllers or screens

# Notes

- **Bootstrap is the composition root** — the only place where concrete classes are instantiated. Feature code imports from `Bootstrap.ts`, never constructs services directly.
- Services call the `@react-native-firebase/*` packages directly for Firestore, Storage, and Functions.
- `FirebaseAuth` is the only library-level class instantiated in Bootstrap because it needs to be a singleton (one `onAuthStateChanged` listener, one MMKV instance).
- Every service has `I{Domain}Service.ts` (interface) + `{Domain}Service.ts` (implementation). Bootstrap exports are typed to interfaces for testability and contract documentation.
- Most services take no constructor args — they get `homesteadId` from `useHomesteadStore`. Only `AuthService(firebaseAuth)` and `HealthService(careService)` have constructor deps.
- RevenueCat is instantiated inside `SubscriptionService`, not exposed from Bootstrap.
