# Instructions

## 1. Firebase Cloud Platform (`src/library/cloudplatform/firebase/`)

### `Firebase.ts` — implements `ICloudPlatform`
- Implements: `getUserId()`, `getDoc()`, `setDoc()`, `updateDoc()`, `collection()`, `collectionGroup()`, `deleteDoc()`, `incrementField()`, `startBatch()`, `commitBatch()`
- Uses `@react-native-firebase/firestore`
- Depends on `Schema.ts` for path resolution

### `FirebaseAuth.ts` — implements `ICloudPlatformAuth`
- Exposes auth state via callback pattern: `onAuthStateChanged(callback: (loggedIn: boolean) => void): () => void`
- The Zustand auth store will call this to stay in sync
- Uses `@react-native-firebase/auth`

### `FirebaseFunctions.ts` — implements `ICloudPlatformFunctions`
- Uses `@react-native-firebase/functions`

### `FirebaseStorage.ts` — implements `ICloudPlatformStorage`
- Uses `@react-native-firebase/storage`

### `FirebaseMessage.ts` — implements `ICloudPlatformMessage`
- Uses `@react-native-firebase/messaging`

### `FirebaseAnalytics.ts` — implements `ICloudPlatformAnalytics`
- Uses `@react-native-firebase/analytics`

### `FirebaseUtil.ts` — utility for Firestore operations

### `Schema.ts` — maps `SchemaType` enum to Firestore document paths

```typescript
export function schemaToDocPaths(schema: SchemaType): string[] {
  switch (schema) {
    case SchemaType.User:                    return ["user"]
    case SchemaType.UserDevice:              return ["user", "device"]
    case SchemaType.Animal:                  return ["animal"]
    case SchemaType.CareEvent:               return ["careEvent"]
    case SchemaType.HealthRecord:            return ["healthRecord"]
    case SchemaType.BreedingRecord:          return ["breedingRecord"]
    case SchemaType.Note:                    return ["note"]
    case SchemaType.WeightLog:              return ["weightLog"]
    case SchemaType.ProductionLog:           return ["productionLog"]
    case SchemaType.AnimalType:              return ["animalType"]
    case SchemaType.AnimalTypeBreed:         return ["animalType", "breed"]
    case SchemaType.AnimalTypeCareTemplate:  return ["animalType", "careTemplate"]
    case SchemaType.AnimalTypeEventTemplate: return ["animalType", "eventTemplate"]
    case SchemaType.Feedback:                return ["feedback"]
    default: throw new Error(`Unknown schema type: ${schema}`)
  }
}
```

**Note:** `UserDevice`, `AnimalTypeBreed`, `AnimalTypeCareTemplate`, and `AnimalTypeEventTemplate` return two-element arrays because they are subcollections. The `ICloudPlatform` implementation uses these paths with parent document IDs to build the full Firestore path (e.g., `animalType/{typeId}/breed/{breedId}`).
