# Goal

Implement the subscription system with RevenueCat integration, three tiers (Free/Pro/Farm), feature gating at the moment of action (paywall triggered when the user tries to use a gated feature), and a subscription management screen. The paywall strategy is to let users experience the app freely and trigger the paywall at the **moment of highest anxiety** — when they have valuable data they're about to lose access to.

## Subscription Gating Utility

### `SubscriptionGate.ts`
```typescript
export type SubscriptionTier = 'free' | 'pro' | 'farm'

export interface GateResult {
  allowed: boolean
  requiredTier: SubscriptionTier
  reason: string
}

export function checkAnimalLimit(currentCount: number, tier: SubscriptionTier): GateResult {
  if (tier !== 'free') return { allowed: true, requiredTier: tier, reason: '' }
  if (currentCount >= 10) return { allowed: false, requiredTier: 'pro', reason: 'Free plan limited to 10 animals. Upgrade to Pro for unlimited.' }
  return { allowed: true, requiredTier: tier, reason: '' }
}

export function checkCareEventLimit(animalEventCount: number, tier: SubscriptionTier): GateResult {
  if (tier !== 'free') return { allowed: true, requiredTier: tier, reason: '' }
  if (animalEventCount >= 3) return { allowed: false, requiredTier: 'pro', reason: 'Free plan limited to 3 care events per animal. Upgrade to Pro for unlimited.' }
  return { allowed: true, requiredTier: tier, reason: '' }
}

export function checkProFeature(tier: SubscriptionTier, featureName: string): GateResult {
  if (tier === 'free') return { allowed: false, requiredTier: 'pro', reason: `${featureName} requires Pro. Upgrade to unlock.` }
  return { allowed: true, requiredTier: tier, reason: '' }
}

export function checkFarmFeature(tier: SubscriptionTier, featureName: string): GateResult {
  if (tier !== 'farm') return { allowed: false, requiredTier: 'farm', reason: `${featureName} requires Farm plan. Upgrade to unlock.` }
  return { allowed: true, requiredTier: tier, reason: '' }
}
```

| Health Records | -- | All types | All types | `checkProFeature` |

## Paywall Screen

### `PaywallController.ts`
```typescript
export function usePaywallController(navigation, route) {
  const { requiredTier, featureName, reason } = route.params ?? {}
  const [offerings, setOfferings] = useState<PurchasesOffering | null>(null)
  const [loading, setLoading] = useState(true)
  const [purchasing, setPurchasing] = useState(false)

  useEffect(() => {
    loadOfferings()
  }, [])

  const loadOfferings = async () => {
    const offerings = await bsSubscriptionService.getOfferings()
    setOfferings(offerings)
    setLoading(false)
  }

  const purchase = async (packageId: string) => {
    setPurchasing(true)
    const result = await bsSubscriptionService.purchasePackage(packageId)
    if (result.success) {
      // Update user subscription in Firestore
      await useUserStore.getState().updateUser({ subscription: requiredTier })
      navigation.goBack()
    }
    setPurchasing(false)
  }

  const restore = async () => {
    setLoading(true)
    await bsSubscriptionService.restorePurchases()
    setLoading(false)
  }

  return { requiredTier, featureName, reason, offerings, loading, purchasing, purchase, restore }
}
```

## Subscription Management Screen

### `SubscriptionController.ts`
```typescript
export function useSubscriptionController(navigation) {
  const { user } = useUserStore()
  const [customerInfo, setCustomerInfo] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    loadSubscriptionInfo()
  }, [])

  const loadSubscriptionInfo = async () => {
    const info = await bsSubscriptionService.getCustomerInfo()
    setCustomerInfo(info)
    setLoading(false)
  }

  const onManageSubscription = () => {
    // Opens native subscription management (App Store / Play Store)
    bsSubscriptionService.openManagement()
  }

  const onUpgrade = () => navigation.navigate('Paywall', {
    requiredTier: user?.subscription === 'free' ? 'pro' : 'farm',
    featureName: 'Full Access',
  })

  return { user, customerInfo, loading, onManageSubscription, onUpgrade }
}
```

### `SubscriptionScreen.tsx`
- **Current plan badge**: Free / Pro / Farm with color
- **Plan details**: What's included in current plan
- **Upgrade button** (if not on Farm): Navigate to Paywall
- **Manage Subscription** button: Opens native store management
- **Restore Purchases** button

## Subscription Service (`src/feature/subscription/service/`)

### `ISubscriptionService.ts`
```typescript
export default interface ISubscriptionService {
  initialize(): Promise<void>
  getOfferings(): Promise<PurchasesOffering | null>
  purchasePackage(packageId: string): Promise<IResult>
  restorePurchases(): Promise<IResult>
  getCustomerInfo(): Promise<CustomerInfo | null>
  openManagement(): void
  getCurrentTier(): SubscriptionTier
}
```

### `SubscriptionService.ts`
- Uses `react-native-purchases` (RevenueCat SDK)
- `initialize()`: Configure Purchases with API key, identify user
- `getOfferings()`: Fetch available subscription offerings
- `purchasePackage()`: Execute purchase, on success update user document `subscription` field
- `restorePurchases()`: Restore previous purchases, update user document
- `getCurrentTier()`: Check RevenueCat entitlements, map to 'free' | 'pro' | 'farm'
- Registered in `Bootstrap.ts`

## Integration with Feature Screens

Each feature screen that needs gating calls the gate utility before allowing the action:

```typescript
// Example: in CreateHealthRecordController
const submit = async () => {
  const gate = checkProFeature(user.subscription, 'Health Records')
  if (!gate.allowed) {
    navigation.navigate('Paywall', {
      requiredTier: gate.requiredTier,
      featureName: 'Health Records',
      reason: gate.reason,
    })
    return
  }
  // proceed with creation
}
```

## Paywall Trigger Points (Moment of Highest Anxiety)

1. User tries to **log a vaccination** → Pro paywall (health records)
2. User tries to **record a breeding event** → Pro paywall (breeding)
3. User tries to **create 11th animal** → Pro paywall (animal limit)
4. User tries to **create 4th care event** for an animal → Pro paywall (care limit)
5. User tries to **log production** → Pro paywall
6. User tries to **export data** → Farm paywall (CSV) or Pro paywall (PDF)

# Notes

- RevenueCat handles all payment processing, receipt validation, and subscription management
- The user's `subscription` field on their Firestore document is the source of truth for client-side gating
- Paywall is triggered at the moment of action, not preemptively — let users see the form, then gate on submit
- This pattern maximizes conversion: user has already committed mentally to the action
- RevenueCat API key must be configured separately for iOS and Android
- Pricing should be set in App Store Connect / Google Play Console, not hardcoded
- Free tier is generous enough to be genuinely useful — 10 animals + 3 care events each covers many small homesteads
