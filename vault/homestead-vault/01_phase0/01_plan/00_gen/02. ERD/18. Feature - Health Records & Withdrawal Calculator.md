# Goal

Build the health records system with all 6 record types (vaccination, medication, deworming, vetVisit, illness, injury), dosage tracking, and the withdrawal period calculator. The withdrawal calculator is the **single highest-anxiety feature** and the **#1 subscription conversion driver**. Food safety depends on it.

## Health Record Detail Screen

### `HealthRecordDetailController.ts`
```typescript
export function useHealthRecordDetailController(navigation, route) {
  const { recordId } = route.params
  const { healthRecords } = useHealthStore()
  const { animals } = useAnimalStore()
  const record = healthRecords.find(r => r.id === recordId)
  const animal = animals.find(a => a.id === record?.animalId)

  // Calculate withdrawal status if applicable
  let withdrawalStatus: WithdrawalResult | null = null
  if (record && record.withdrawalPeriodDays > 0) {
    withdrawalStatus = calculateWithdrawal(record.date, record.withdrawalPeriodDays, record.withdrawalType)
  }
  if (record && record.dewormingWithdrawalDays > 0) {
    withdrawalStatus = calculateWithdrawal(record.date, record.dewormingWithdrawalDays, record.dewormingWithdrawalType)
  }

  return { record, animal, withdrawalStatus }
}
```

## Health Service (`src/feature/health/service/`)

### `IHealthService.ts`
```typescript
export default interface IHealthService {
  fetchHealthRecordsByAnimal(userId: string, animalId: string): Promise<HealthRecord[]>
  fetchAllWithdrawalRecords(userId: string): Promise<HealthRecord[]>
  createHealthRecord(record: HealthRecord, photoUri?: string): Promise<IResult>
}
```

### `HealthService.ts`
- `fetchHealthRecordsByAnimal`: query `healthRecord` where `userId == uid` AND `animalId == id` AND `admin.deleted == false`, order by `date` DESC
- `fetchAllWithdrawalRecords`: query `healthRecord` where `userId == uid` AND `admin.deleted == false` AND `withdrawalPeriodDays > 0`
  - Client-side: also include records where `dewormingWithdrawalDays > 0`
- `createHealthRecord`: setDoc + photo upload if provided
  - **Side effect**: If `recordType == 'vaccination'` AND `vaccineNextDueDate` is set, auto-create a CareEvent for that animal on that date (name: "Vaccination: {name}", type: 'careSingle')
- Registered in `Bootstrap.ts`

# Key Design Decisions

1. **No hardcoded medication database** — farmers use off-label drugs frequently. The user enters withdrawal period manually.
2. **Multiple overlapping withdrawals** — one animal can have active meat withdrawal AND active milk withdrawal simultaneously
3. **Withdrawal type matters** — an egg farmer with Corid (0-day) vs ivermectin (14+ day) has very different timelines
4. **Dosage context always displayed together** — amount + unit + route + frequency are always shown as a group, never separated
5. **Safety-critical UI** — active withdrawal status uses unmistakable red, cleared uses unmistakable green. No ambiguity.

# Subscription Gating

- **Free tier**: No access to health records. Attempting to create one triggers Pro paywall.
- **Pro/Farm tier**: Full access to all 6 record types and withdrawal calculator.

# Notes

- This is the #1 conversion driver — the moment a user tries to log a vaccination with a withdrawal period, the data is too valuable to abandon
- The withdrawal calculator surfaces on: Today Dashboard, Animal Detail banner, Health Record Detail
- Multiple overlapping withdrawals per animal are common (e.g., Penicillin with 4-day milk AND 8-day meat)
- Vaccination with `vaccineNextDueDate` creates a one-time care reminder — connects health records to the care system
- Off-label use means the user determines the withdrawal period themselves — the app is intentionally flexible
