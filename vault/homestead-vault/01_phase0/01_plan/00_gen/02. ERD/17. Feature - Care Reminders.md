# Goal

Build the Care tab with cross-animal care schedule, care event CRUD, recurring care auto-creation on completion, and due date status coloring. Care reminders are the backbone of daily app usage — they turn the app into an indispensable daily checklist.

## Care List Screen (Care Tab)

### `CareListController.ts`
```typescript
export function useCareListController(navigation) {
  const { careEvents, fetchCareEvents, loading } = useCareStore()
  const { animals } = useAnimalStore()
  const [filterAnimalId, setFilterAnimalId] = useState<string | null>(null)
  const [refreshing, setRefreshing] = useState(false)

  useFocusEffect(useCallback(() => {
    fetchCareEvents()
  }, []))

  // Filter incomplete events only
  const incompleteEvents = careEvents.filter(e => !e.completedDate)

  // Apply animal filter
  const filteredEvents = filterAnimalId
    ? incompleteEvents.filter(e => e.animalId === filterAnimalId)
    : incompleteEvents

  // Group by status
  const overdue = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'OVERDUE')
  const dueToday = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'DUE_TODAY')
  const upcoming = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'UPCOMING')
  const future = filteredEvents.filter(e => getCareStatus(e.dueDate).status === 'FUTURE')

  // Helper to get animal name
  const getAnimalName = (animalId: string) => animals.find(a => a.id === animalId)?.name ?? ''

  const onEventPress = (eventId: string) => navigation.navigate('CareEventDetail', { eventId })
  const onCreateEvent = () => navigation.navigate('CreateCareEvent', { animalId: filterAnimalId ?? '' })

  const onRefresh = async () => {
    setRefreshing(true)
    await fetchCareEvents()
    setRefreshing(false)
  }

  return {
    overdue, dueToday, upcoming, future, loading, refreshing,
    animals, filterAnimalId, setFilterAnimalId, getAnimalName,
    onEventPress, onCreateEvent, onRefresh,
  }
}
```

## Care Event Detail Screen

### `CareEventDetailController.ts`
```typescript
export function useCareEventDetailController(navigation, route) {
  const { eventId } = route.params
  const { careEvents, completeCareEvent } = useCareStore()
  const { animals } = useAnimalStore()
  const event = careEvents.find(e => e.id === eventId)
  const animal = animals.find(a => a.id === event?.animalId)

  const status = event ? getCareStatus(event.dueDate) : null
  const isComplete = !!event?.completedDate

  const onComplete = async () => {
    await completeCareEvent(eventId)
    navigation.goBack()
  }

  return { event, animal, status, isComplete, onComplete }
}
```

  useEffect(() => {
    if (templateId) {
      const template = /* find template */
      if (template) {
        setName(template.name)
        setType(template.type)
        setCycle(template.cycle)
        setContactName(template.contactName)
        setContactPhone(template.contactPhone)

## Care Service (`src/feature/care/service/`)

### `ICareService.ts`
```typescript
export default interface ICareService {
  fetchCareEventsByUser(userId: string): Promise<CareEvent[]>
  fetchCareEventsByAnimal(userId: string, animalId: string): Promise<CareEvent[]>
  createCareEvent(event: CareEvent): Promise<IResult>
  completeCareEvent(event: CareEvent): Promise<IResult>
}
```

### `CareService.ts`
- `fetchCareEventsByUser`: query `careEvent` where `userId == uid` AND `admin.deleted == false`, order by `dueDate`
- `completeCareEvent`: **Critical business logic**:
  1. Set `completedDate` to today's ISO string
  2. If `type == 'careRecurring'` AND `createdNextRecurringEvent == false`:
     a. Create new careEvent with `dueDate = completedDate + cycle` (using `calculateNextDueDate()`)
     b. Copy: name, type, cycle, animalId, userId, templateId, contactName, contactPhone
     c. New event has: empty completedDate, createdNextRecurringEvent=false
  3. Set `createdNextRecurringEvent = true` on the completed event
  4. Use batch write for atomicity
- **Next due date calculated from completion date, NOT original due date** — prevents cascading drift
- One-time events (`careSingle`) don't create next occurrences
- Registered in `Bootstrap.ts`

# Subscription Gating

- **Free tier**: 3 care events per animal. On create, check count for the target animal.
- **Pro/Farm tier**: Unlimited care events.

# Notes

- Completing a recurring care event is the most important business logic in the app
- Always calculate next due date from `completedDate`, never from the original `dueDate`
- Template updates don't retroactively change existing care events
- The Cloud Function `onCareEventComplete` trigger is a safety net for the recurring event creation
- Care events from templates carry the `templateId` reference but are independent copies
- The cross-animal care view (Care tab) is the second-most-used screen after the Today Dashboard
