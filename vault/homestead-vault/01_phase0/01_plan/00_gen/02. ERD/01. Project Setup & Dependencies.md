# Goal

Initialize the Homestead React Native project with the correct folder structure, dependencies, and build configuration. Mirror the Ephesus/Antioch architecture but use `Uniwind` (Tailwind CSS for React Native, drop-in NativeWind replacement) for styling and `Zustand` for state management.

# References

- `HomesteadApp/package.json` - Current bare RN project
- `~/Projects/ephesus/EphesusApp/package.json` - Reference dependency list
- `~/Projects/ephesus/EphesusApp/src/` - Reference folder structure

# Instructions

1. Create the `HomesteadApp` React Native project using the React Native CLI (bare workflow, not Expo):
   ```bash
   npx @react-native-community/cli init HomesteadApp
   ```

2. Update `HomesteadApp/package.json` with required dependencies:
   - **Firebase**: `@react-native-firebase/app`, `auth`, `firestore`, `functions`, `messaging`, `storage`, `crashlytics`, `analytics`
   - **Navigation**: `@react-navigation/native`, `native-stack`, `bottom-tabs`, `drawer`
   - **State**: `zustand`
   - **Styling**: `uniwind`, `tailwindcss`
   - **Storage**: `react-native-mmkv`
   - **UI**: `react-native-screens`, `react-native-safe-area-context`, `react-native-gesture-handler`, `react-native-reanimated`, `@react-native-vector-icons/material-design-icons`
   - **Pickers**: `@react-native-community/datetimepicker`, `react-native-image-picker`, `react-native-image-resizer`
   - **Subscriptions**: `react-native-purchases` (RevenueCat)
   - **QR**: `react-native-camera`, `react-native-qrcode-svg` (for QR Barn Mode)
   - **Utility**: `react-native-uuid`, `date-fns`
   - **Dev**: `@types/jest`, `@types/react`, `typescript`, `jest`, `prettier`, `eslint`

3. Configure Uniwind (`global.css` with TailwindCSS v4, metro config with `withUniwindConfig`)

4. Create the `src/` folder structure inside `HomesteadApp/`:

```
src/
  Bootstrap.ts
  components/
  core/
    plugin/
    service/
  feature/
    auth/
    loading/
    home/
    animal/
    care/
    health/
    breeding/
    production/
    notes/
    profile/
    customization/
    subscription/
    debug/
  library/
    cloudplatform/
      firebase/
    log/
    storage/
      mmkv/
  navigation/
  schema/
    object/
    type/
    user/
    animal/
    care/
    health/
    breeding/
    production/
    notes/
    weight/
    animalType/
    feedback/
  store/
  theme/
  types/
  util/
```

5. Update `App.tsx` to use `SafeAreaProvider` and `RootNavigation` (no PaperProvider — use Uniwind instead)
6. Update `tsconfig.json` with path aliases if needed
7. Run `npm install`, `cd ios && pod install`

# Key Architecture Decisions

- **No** `react-native-paper` — all UI via Uniwind/Tailwind classes
- **No** `rxjs` — replaced by Zustand stores for reactive state
- **Add** `zustand` for state management
- **Add** `uniwind` + `tailwindcss` for styling
- **Add** `date-fns` for date operations (tree-shakeable, replaces moment)
- **Add** `react-native-purchases` for RevenueCat subscription management
- React Native `0.84+` (latest stable)

# Notes

- The project structure mirrors Ephesus exactly — same architecture, different domain
- Firebase project must be created separately and `google-services.json` / `GoogleService-Info.plist` placed in the appropriate directories
- RevenueCat requires separate setup in App Store Connect and Google Play Console
