# Goal

Build reusable UI components in `src/components/` that are shared across multiple features. All styled with Uniwind (Tailwind CSS for React Native, drop-in NativeWind replacement). These are the building blocks for the feature screens. Designed for barn-friendly use — large touch targets, high contrast status indicators, and clear visual hierarchy.

# Instructions

## 1. Centralized Component Directory

**All shared/reusable UI components live in `src/components/`.** This is the single source of truth for UI building blocks. Feature screens import from here — never duplicate component code in feature folders.


## 2. Component Specifications

### `ScreenContainer.tsx`
- Wraps `SafeAreaView` with consistent padding and background color
- Uniwind classes: `className="flex-1 bg-background"`

### `AnimalCard.tsx`
- Displays: Photo thumbnail, name, breed, age, state badge
- Optional: active withdrawal indicator (red dot), upcoming care indicator
- Props: `animal: Animal`, `onPress`, `showStatus?: boolean`
- Uniwind: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm flex-row"`

### `CareEventCard.tsx`
- Displays: Event name, animal name, due date, status color bar (left border)
- Status colors: overdue (red left border), due-today (amber), upcoming (green), future (gray)
- Action: "Mark Complete" button
- Props: `event: CareEvent`, `animalName: string`, `onComplete`, `onPress`

### `HealthRecordCard.tsx`
- Displays: Record type icon, name, date, withdrawal status badge (if applicable)
- Props: `record: HealthRecord`, `onPress`

### `BreedingCard.tsx`
- Displays: Dam name, sire name, breeding date, countdown/progress bar, status
- Active: shows days remaining + progress bar
- Completed: shows birth outcome summary
- Props: `record: BreedingRecord`, `damName: string`, `onPress`

### `ProductionCard.tsx`
- Displays: Production type icon, quantity + unit, date, animal/group name
- Props: `log: ProductionLog`, `onPress`

### `NoteCard.tsx`
- Displays: Text snippet, tags as colored chips, date, photo thumbnail if present
- Props: `note: Note`, `onPress`

### `DashboardAlertCard.tsx`
- Displays: Alert type icon, title, subtitle, color-coded urgency
- Variants: withdrawal active (red), breeding due soon (amber), overdue care (red)
- Props: `type: 'withdrawal' | 'breeding' | 'care'`, `title`, `subtitle`, `urgency: 'high' | 'medium' | 'low'`, `onPress`

### `PrimaryButton.tsx`
- Full-width button with primary color
- Uniwind: `className="bg-primary rounded-xl py-3 items-center"`
- Props: `title: string`, `onPress`, `loading?: boolean`, `disabled?: boolean`
- Minimum height 48px for barn-friendly touch targets

### `FloatingActionButton.tsx`
- Positioned bottom-right
- Circle with "+" icon
- Props: `onPress`

### `Avatar.tsx`
- Circular image with fallback initials (or animal icon for animal avatars)
- Sizes: `sm` (32), `md` (44), `lg` (64)
- Props: `uri?: string`, `name: string`, `size: 'sm' | 'md' | 'lg'`

### `TextInput.tsx`
- Styled text input with label and error state
- Uniwind: `className="border border-border-light rounded-lg px-4 py-3 text-base"`
- Props: `label`, `value`, `onChangeText`, `error?`, `secureTextEntry?`, `placeholder?`, `multiline?`

### `DatePicker.tsx`
- Wraps `@react-native-community/datetimepicker`
- Shows formatted date in a tappable field, opens native picker on press
- Props: `label`, `value: string` (ISO 8601), `onChange: (date: string) => void`, `mode?: 'date' | 'datetime'`

### `NumberInput.tsx`
- Numeric input with +/- stepper buttons on sides
- Props: `label`, `value: number`, `onChange`, `min?`, `max?`, `step?`, `unit?: string`

### `DropdownSelect.tsx`
- Tappable field that opens `SelectDialog` with options
- Props: `label`, `value`, `options: { label, value }[]`, `onChange`, `placeholder?`

### `SearchBar.tsx`
- Text input with search icon and clear button
- Props: `value`, `onChangeText`, `placeholder?`

### `EmptyState.tsx`
- Centered message with icon for empty lists
- Props: `icon: string`, `title: string`, `subtitle?: string`

### `LoadingIndicator.tsx`
- Centered ActivityIndicator with optional message
- Props: `message?: string`

### `Badge.tsx`
- Small colored badge for counts and labels
- Props: `label: string`, `variant: 'primary' | 'secondary' | 'accent'`

### `StatusBadge.tsx`
- Color-coded status indicator specifically for care/health statuses
- Variants: `overdue` (red), `dueToday` (amber), `upcoming` (green), `future` (gray), `active` (red), `clear` (green)
- Props: `status: string`, `label?: string`

### `ProgressBar.tsx`
- Horizontal progress bar for gestation countdown
- Props: `progress: number` (0-100), `color?: string`, `label?: string`

### `PhotoPicker.tsx`
- Image display + tap to pick from camera/gallery
- Uses `react-native-image-picker` and `react-native-image-resizer`
- Props: `uri?: string`, `onPick: (uri: string) => void`, `size?: 'sm' | 'md' | 'lg'`

### `TagChip.tsx`
- Small colored rounded tag
- Color mapped to tag name (Health=red, Behavior=blue, Breeding=purple, Feed=orange, Production=green, General=gray)
- Props: `tag: string`, `onPress?`

### `TagSelector.tsx`
- Row of `TagChip` components with multi-select capability
- Props: `tags: string[]`, `selected: string[]`, `onToggle: (tag: string) => void`

### `AppDialog.tsx` — Custom Dialog
- Built on React Native `Modal` with `transparent` + `animationType="fade"`
- Overlay: semi-transparent black background, tap to dismiss
- Content card: white rounded card centered on screen
- Props: `visible: boolean`, `onDismiss`, `title?: string`, `children`, `actions?: ReactNode`

### `ConfirmDialog.tsx`
- Wraps `AppDialog` with standard confirm/cancel pattern
- Props: `visible`, `onConfirm`, `onCancel`, `title`, `message`, `confirmLabel?`, `cancelLabel?`, `destructive?: boolean`

### `SelectDialog.tsx`
- Wraps `AppDialog` with a list of selectable items
- Props: `visible`, `onDismiss`, `title`, `items: { label, value }[]`, `onSelect`, `selectedValue?`

### `ThemeButton.tsx`
- Variant-based button
- Props: `variant: 'primary' | 'secondary' | 'tertiary'`, `buttonStyle: 'outline' | 'solid'`, `title`, `onPress`, `disabled?`, `loading?`

## 4. Uniwind Styling Conventions

- Use `className` prop on all RN components
- Avoid inline `style` objects unless dynamic values are needed
- Common patterns:
  - Cards: `className="bg-surface rounded-xl p-4 mb-3 shadow-sm"`
  - Text primary: `className="text-base text-gray-900"`
  - Text secondary: `className="text-sm text-gray-500"`
  - Dividers: `className="h-px bg-border-light my-2"`
  - Status borders: `className="border-l-4 border-danger"` (for overdue care)

# Notes

- All components should be functional components with TypeScript props
- No `react-native-paper` components — everything is built from RN primitives + Uniwind
- Components should be presentational (no state management or service calls)
- Pass callbacks via props; the parent screen/controller handles logic
- Minimum touch target size: 44px height — barn-friendly design
- Status colors must be consistent across all components: red=urgent/active, amber=warning/today, green=healthy/clear, gray=neutral/future
