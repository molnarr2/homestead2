# Goal

Define the abstraction layer interfaces in `src/core/plugin/`. These interfaces decouple business logic from third-party libraries, making it possible to swap implementations (e.g., Firebase for another backend) without touching feature code.

# Instructions

1. Create `src/core/plugin/ICloudPlatform.ts`:
   - `WhereFilterOp`, `OrderByDirection` types
   - `IWhereClause`, `ISortClause` interfaces
   - `ICloudPlatform` interface with: `getUserId()`, `getDoc()`, `setDoc()`, `updateDoc()`, `collection()`, `collectionGroup()`, `deleteDoc()`, `incrementField()`, `startBatch()`, `commitBatch()`
   - Import `SchemaType` and `CloudDocumentData` from the schema layer

2. Create `src/core/plugin/ICloudPlatformAuth.ts`:
   - Properties: `currentUserId`, `isAnonymous`, `isLoggedIn`
   - Methods: `createUserWithEmailAndPassword()`, `signInWithEmailAndPassword()`, `waitUntilAuthenticated()`, `signOut()`, `sendPasswordResetEmail()`, `deleteAuthAccount()`
   - Use a simple callback pattern instead of rxjs: `onAuthStateChanged(callback: (loggedIn: boolean) => void): () => void`

3. Create `src/core/plugin/ICloudPlatformFunctions.ts`:
   - `callFunction(name, data)` for invoking Firebase Cloud Functions

4. Create `src/core/plugin/ICloudPlatformMessage.ts` — for push notifications:
   - `requestPermission()`, `getToken()`, `onMessage(callback)`

5. Create `src/core/plugin/ICloudPlatformStorage.ts` — for file uploads:
   - `uploadFile(path, uri)`, `getDownloadUrl(path)`

6. Create `src/core/plugin/ICloudPlatformAnalytics.ts` — for analytics:
   - `logEvent(name, params)`, `setUserId(id)`

7. Create `src/core/plugin/ILocalStorage.ts`:
   - `setString()`, `setBoolean()`, `setNumber()`
   - `getString()`, `getBoolean()`, `getNumber()`
   - `contains()`, `delete()`, `clearAll()`

# Key Difference from Ephesus

- **No `rxjs`** — auth state is managed via Zustand. The auth plugin only needs synchronous getters and async methods. The Zustand auth store will subscribe to Firebase auth state changes directly.
- Interfaces are identical to Ephesus — same abstraction pattern, same method signatures.

# Schema Dependency

These interfaces depend on `src/schema/SchemaDefinition.ts` which defines `SchemaType` enum and `CloudDocumentData` interface. The `SchemaType` enum for Homestead includes:

```typescript
export enum SchemaType {
  User,
  UserDevice,
  Animal,
  CareEvent,
  HealthRecord,
  BreedingRecord,
  Note,
  WeightLog,
  ProductionLog,
  AnimalType,
  AnimalTypeBreed,
  AnimalTypeCareTemplate,
  AnimalTypeEventTemplate,
  Feedback,
}
```

**Note:** `AnimalTypeBreed`, `AnimalTypeCareTemplate`, and `AnimalTypeEventTemplate` are subcollections under `AnimalType`, matching the Ephesus pattern for subcollections (e.g., `animalType/{id}/breed/{breedId}`).
