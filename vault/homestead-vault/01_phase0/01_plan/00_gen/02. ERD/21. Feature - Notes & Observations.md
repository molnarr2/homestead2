# Goal

Build the notes system with quick timestamped entries per animal, tag-based categorization, optional photo attachment, and display on the Animal Detail timeline. Notes are the simplest per-animal data type — they capture the observations and context that don't fit neatly into structured records.

## Create Note Screen

### `CreateNoteController.ts`
```typescript
export function useCreateNoteController(navigation, route) {
  const { animalId } = route.params
  const { createNote } = useNoteStore()
  const userId = useAuthStore().userId

  const [text, setText] = useState('')
  const [tags, setTags] = useState<NoteTag[]>([])
  const [photoUri, setPhotoUri] = useState('')
  const [loading, setLoading] = useState(false)

  const toggleTag = (tag: NoteTag) => {
    setTags(prev =>
      prev.includes(tag) ? prev.filter(t => t !== tag) : [...prev, tag]
    )
  }

  const submit = async () => {
    if (!text.trim()) return
    setLoading(true)
    await createNote({ userId, animalId, text, tags, photoUri })
    setLoading(false)
    navigation.goBack()
  }

  return { text, setText, tags, toggleTag, photoUri, setPhotoUri, loading, submit }
}
```

### `CreateNoteScreen.tsx`
- **Text input**: Large multiline TextInput with placeholder "What did you observe?"
- **Tag selector** (`TagSelector`): Row of selectable tag chips
  - Health (red), Behavior (blue), Breeding (purple), Feed (orange), Production (green), General (gray)
  - Multi-select: tap to toggle
- **Photo attachment** (`PhotoPicker`): Optional photo from camera or gallery
- Save button

## Note Detail Screen

### `NoteDetailController.ts`
```typescript
export function useNoteDetailController(navigation, route) {
  const { noteId } = route.params
  const { notes, deleteNote } = useNoteStore()
  const { animals } = useAnimalStore()
  const note = notes.find(n => n.id === noteId)
  const animal = animals.find(a => a.id === note?.animalId)

  const onDelete = async () => {
    await deleteNote(noteId)
    navigation.goBack()
  }

  return { note, animal, onDelete }
}
```

### `NoteDetailScreen.tsx`
- Animal name at top (tappable → AnimalDetail)
- Full text content
- Tags displayed as `TagChip` components
- Photo (full-width if present)
- Date + relative time ("2 hours ago")
- Delete button (with ConfirmDialog)

## Note Display on Animal Detail

Notes are rendered in the `AnimalNotesTab.tsx` component (defined in doc 16):
- List of `NoteCard` components sorted by `admin.created_at` DESC
- Each card shows: text snippet (truncated), tags, date, photo thumbnail if present
- Tapping opens `NoteDetailScreen`

## Note Service (`src/feature/notes/service/`)

### `INoteService.ts`
```typescript
export default interface INoteService {
  fetchNotesByAnimal(userId: string, animalId: string): Promise<Note[]>
  createNote(note: Note, photoUri?: string): Promise<IResult>
  deleteNote(noteId: string): Promise<IResult>   // soft-delete
}
```

### `NoteService.ts`
- `fetchNotesByAnimal`: query `note` where `userId == uid` AND `animalId == id` AND `admin.deleted == false`, order by `admin.created_at` DESC
- `createNote`: setDoc + photo upload if provided. Photo stored at `animals/{animalId}/notes/{noteId}.jpg`
- `deleteNote`: soft-delete via `admin.deleted = true`
- Registered in `Bootstrap.ts`

- **Pro/Farm tier**: Full access.

# Notes

- Notes are the simplest per-animal record type — quick capture of observations
- They supplement structured records (health, breeding, care) with free-form context
- Tags enable filtering on the Animal Detail timeline
- Photo attachment is optional but useful for documenting visible symptoms, injuries, or condition
- Notes display in the Animal Detail timeline alongside other record types
- Soft-delete only — notes are never hard-deleted
