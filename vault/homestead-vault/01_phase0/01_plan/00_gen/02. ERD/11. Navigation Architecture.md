# Goal

Set up the navigation structure using React Navigation. Follows the Ephesus/Antioch pattern with a root stack navigator, plus a **slide-out side drawer menu** accessed via the user's avatar in the top-left of the main screen. Four bottom tabs: Home, Animals, Production, Care.

# Instructions

## 1. Root Navigation (`src/navigation/RootNavigation.tsx`)

Three navigation states driven by Zustand `authStore.isLoggedIn`:

- **`null`** → Show `LoadingScreen` (checking auth state)
- **`false`** → Show auth stack (Login, Register)
- **`true`** → Show drawer navigator wrapping the main app

```typescript
export type RootStackParamList = {
  Loading: undefined
  Login: undefined
  Register: undefined
  SpeciesSelection: undefined
  DrawerMain: undefined

  // Animal screens
  AnimalDetail: { animalId: string }
  CreateAnimal: { animalTypeId?: string }
  EditAnimal: { animalId: string }

  // Care screens
  CareEventDetail: { eventId: string }
  CreateCareEvent: { animalId: string; templateId?: string }

  // Health screens
  HealthRecordDetail: { recordId: string }
  CreateHealthRecord: { animalId: string; recordType?: HealthRecordType }

  // Breeding screens
  BreedingRecordDetail: { recordId: string }
  CreateBreedingRecord: { animalId: string }

  // Production screens
  CreateProductionLog: { animalId?: string; groupName?: string }

  // Notes screens
  CreateNote: { animalId: string }

  // Weight screens
  CreateWeightLog: { animalId: string }

  // Settings screens
  Profile: undefined
  EditProfile: undefined

```

- Use `createNativeStackNavigator` with `headerShown: false`
- Subscribe to `useAuthStore()` for the `isLoggedIn` state

## 2. Drawer Navigation (`src/navigation/DrawerNavigator.tsx`)

Use `@react-navigation/drawer` to wrap the main tab screen:

```typescript
import { createDrawerNavigator } from '@react-navigation/drawer'

const Drawer = createDrawerNavigator()

export function DrawerNavigator() {
  return (
    <Drawer.Navigator
      drawerContent={(props) => <SideMenu {...props} />}
      screenOptions={{ headerShown: false, drawerType: 'slide' }}
    >
      <Drawer.Screen name="MainTabs" component={MainScreen} />
    </Drawer.Navigator>
  )
}
```

## 3. Side Menu (`src/navigation/SideMenu.tsx`)

The custom drawer content. Accessed by tapping the user avatar in the top-left of the main screen header.

### Layout:
- **Header**: User avatar (large), name, email, subscription badge (Free/Pro/Farm)
- **Divider**
- **Menu Items**:
  - **Profile** — navigates to profile/account screen
  - **Subscription** — navigates to subscription management (shows current tier)
  - **Customization** — navigates to animal types, breeds, care templates management
  - **Divider**
  - **Settings** — app settings (notifications, units preferences)
  - **Send Feedback** — in-app feedback form
  - **Divider**
  - **Debug Theme** — navigates to the debug/component showcase screen (dev/debug item)
  - **Divider**
  - **Logout** — calls `bsAuthService.signout()` and `resetAllStores()`

### Controller (`src/navigation/SideMenuController.ts`):
```typescript
export function useSideMenuController() {
  const { user } = useUserStore()

  const logout = () => {
    bsAuthService.signout()
    resetAllStores()
  }

  return { user, logout }
}
```

## 6. Controller Pattern

Each screen pair follows the Controller + Screen pattern:
- `{Feature}Controller.ts` — custom hook with business logic
- `{Feature}Screen.tsx` — pure UI component that uses the controller hook

```typescript
// HomeController.ts
export function useHomeController(navigation: HomeNavigationProp) {
  const { careEvents, fetchCareEvents } = useCareStore()
  const { user } = useUserStore()

  useFocusEffect(
    useCallback(() => {
      fetchCareEvents()
    }, [])
  )

  return { careEvents, user }
}
```

# Notes

- Detail/create screens are on the root stack, not nested in tabs — they push over the tab bar
- The drawer slides out from the left — standard mobile pattern
- 4 tabs (not 5) keeps the bottom bar clean
- Animal Detail is the hub — health records, breeding, care, notes, weight all accessed from there
- Production gets its own tab because daily logging is a ritual action
- Care gets its own tab because the cross-animal view is the 2nd most important screen
- Debug screen is always visible in the side menu (useful for development)
- QR Barn Mode will be triggered from the Home screen FAB or a dedicated scan button (future feature doc)
