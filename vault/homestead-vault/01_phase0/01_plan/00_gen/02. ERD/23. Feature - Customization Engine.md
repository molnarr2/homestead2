# Goal

Build the customization engine accessible from the side menu. Users manage their own animal types, breeds (with optional gestation day overrides), care templates, and event templates. Starter playbooks are seeded during onboarding based on selected species. This is what makes the app feel personal — it adapts to each homesteader's specific animals and routines.

## Animal Type Detail Screen

### `AnimalTypeDetailController.ts`
```typescript
export function useAnimalTypeDetailController(navigation, route) {
  const { animalTypeId } = route.params
  const { animalTypes, breeds, careTemplates, eventTemplates,
          fetchBreeds, fetchCareTemplates, fetchEventTemplates } = useAnimalTypeStore()

  const animalType = animalTypes.find(t => t.id === animalTypeId)

  useEffect(() => {
    fetchBreeds(animalTypeId)
    fetchCareTemplates(animalTypeId)
    fetchEventTemplates(animalTypeId)
  }, [animalTypeId])

  const typeBreeds = breeds[animalTypeId] ?? []
  const typeCareTemplates = careTemplates[animalTypeId] ?? []
  const typeEventTemplates = eventTemplates[animalTypeId] ?? []

  const onEditType = () => navigation.navigate('EditAnimalType', { animalTypeId })
  const onAddBreed = () => navigation.navigate('EditBreed', { animalTypeId })
  const onEditBreed = (breedId: string) => navigation.navigate('EditBreed', { animalTypeId, breedId })
  const onAddCareTemplate = () => navigation.navigate('EditCareTemplate', { animalTypeId })
  const onEditCareTemplate = (templateId: string) =>
    navigation.navigate('EditCareTemplate', { animalTypeId, templateId })

  return {
    animalType, typeBreeds, typeCareTemplates, typeEventTemplates,
    onEditType, onAddBreed, onEditBreed, onAddCareTemplate, onEditCareTemplate,
  }
}
```

### `AnimalTypeDetailScreen.tsx`
- **Header**: Animal type name + edit button
- **Colors section**: Display configured color options (used for animal color selection)
- **Breeds section** (`BreedList`):
  - List of breeds with name and gestation days override (if set)
  - "+" button to add breed
  - Each breed tappable → edit
- **Care Templates section** (`CareTemplateList`):
  - List of care templates with name, type (recurring/single), cycle days
  - "+" button to add template
  - Each template tappable → edit
- **Event Templates section** (`EventTemplateList`):
  - List of event templates with name
  - "+" button to add template

## Edit Animal Type Screen

### `EditAnimalTypeController.ts`
```typescript
export function useEditAnimalTypeController(navigation, route) {
  const { animalTypeId } = route.params ?? {}
  const { animalTypes, createAnimalType } = useAnimalTypeStore()
  const userId = useAuthStore().userId
  const existing = animalTypeId ? animalTypes.find(t => t.id === animalTypeId) : null

  const [name, setName] = useState(existing?.name ?? '')
  const [colors, setColors] = useState<string[]>(existing?.colors ?? [])
  const [newColor, setNewColor] = useState('')
  const [loading, setLoading] = useState(false)

  const addColor = () => {
    if (newColor.trim() && !colors.includes(newColor.trim())) {
      setColors([...colors, newColor.trim()])
      setNewColor('')
    }
  }

  const removeColor = (color: string) => setColors(colors.filter(c => c !== color))

  const save = async () => {
    if (!name.trim()) return
    setLoading(true)
    if (existing) {
      // update
    } else {
      await createAnimalType({ userId, name, colors })
    }
    setLoading(false)
    navigation.goBack()
  }

  return { name, setName, colors, newColor, setNewColor, addColor, removeColor, loading, save, isEditing: !!existing }
}
```

## AnimalType Service (`src/feature/customization/service/`)

### `IAnimalTypeService.ts`
```typescript
export default interface IAnimalTypeService {
  fetchAnimalTypes(userId: string): Promise<AnimalType[]>
  createAnimalType(type: AnimalType): Promise<IResult>
  updateAnimalType(type: AnimalType): Promise<IResult>
  fetchBreeds(animalTypeId: string): Promise<Breed[]>
  createBreed(animalTypeId: string, breed: Breed): Promise<IResult>
  fetchCareTemplates(animalTypeId: string): Promise<CareTemplate[]>
  createCareTemplate(animalTypeId: string, template: CareTemplate): Promise<IResult>
  fetchEventTemplates(animalTypeId: string): Promise<EventTemplate[]>
  seedStarterPlaybook(userId: string, species: string): Promise<void>
}
```

### `AnimalTypeService.ts`
- Uses subcollection paths: `animalType/{typeId}/breed/{breedId}`, etc.
- `seedStarterPlaybook`: Creates AnimalType + breeds + careTemplates from `StarterPlaybooks.ts` data
  - Uses batch write for atomicity
- Registered in `Bootstrap.ts`

# Notes

- Starter playbooks are seeded once during onboarding — the data is copied into the user's Firestore documents
- Template updates do NOT retroactively change existing care events — events are independent copies
- Breeds can optionally override the species default gestation days
- Colors array on AnimalType is used when creating/editing an animal (color dropdown options)
- Users can create entirely custom animal types beyond the defaults (e.g., "Bees", "Fish")
- The customization engine is what makes the app feel like "my farm app" vs "a generic template"
