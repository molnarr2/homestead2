# Goal

Build the Production tab with daily logging for eggs, milk, fiber, honey, meat, and other production types. Supports per-animal or per-group (flock/herd) logging with daily/weekly/monthly aggregation and trend visibility. Production logging is a daily ritual — the tab must support one-tap quick entry.

## Production List Screen (Production Tab)

### `ProductionListController.ts`
```typescript
export function useProductionListController(navigation) {
  const { productionLogs, fetchProductionLogs, loading } = useProductionStore()
  const [selectedType, setSelectedType] = useState<ProductionType | 'all'>('all')
  const [refreshing, setRefreshing] = useState(false)
  const today = todayIso()

  useFocusEffect(useCallback(() => {
    fetchProductionLogs()
  }, []))

  const filteredLogs = selectedType === 'all'
    ? productionLogs
    : productionLogs.filter(l => l.productionType === selectedType)

  // Aggregations using ProductionUtility
  const dailySummary = aggregateProduction(filteredLogs, today)

  // Recent logs (last 7 days)
  const recentLogs = filteredLogs
    .filter(l => daysBetween(l.date, today) <= 7)
    .sort((a, b) => b.date.localeCompare(a.date))

  const onCreateLog = (type?: ProductionType) =>
    navigation.navigate('CreateProductionLog', { type })

  const onRefresh = async () => {
    setRefreshing(true)
    await fetchProductionLogs()
    setRefreshing(false)
  }

  return {
    dailySummary, recentLogs, selectedType, setSelectedType,
    loading, refreshing, onRefresh, onCreateLog,
  }
}
```

## Create Production Log Screen

### `CreateProductionLogController.ts`
```typescript
export function useCreateProductionLogController(navigation, route) {
  const { animalId: initialAnimalId, groupName: initialGroup, type: initialType } = route.params ?? {}
  const { createProductionLog } = useProductionStore()
  const { animals } = useAnimalStore()
  const userId = useAuthStore().userId

  const [productionType, setProductionType] = useState<ProductionType>(initialType ?? 'eggs')
  const [quantity, setQuantity] = useState(0)
  const [unit, setUnit] = useState(getDefaultUnit(initialType ?? 'eggs'))
  const [date, setDate] = useState(todayIso())
  const [animalId, setAnimalId] = useState(initialAnimalId ?? '')
  const [groupName, setGroupName] = useState(initialGroup ?? '')
  const [notes, setNotes] = useState('')
  const [logMode, setLogMode] = useState<'animal' | 'group'>(initialAnimalId ? 'animal' : 'group')
  const [loading, setLoading] = useState(false)

  // Update default unit when type changes
  useEffect(() => {
    setUnit(getDefaultUnit(productionType))
  }, [productionType])

  const submit = async () => {
    if (quantity <= 0) return
    setLoading(true)
    await createProductionLog({
      userId, productionType, quantity, unit, date, notes,
      animalId: logMode === 'animal' ? animalId : '',
      groupName: logMode === 'group' ? groupName : '',
    })
    setLoading(false)
    navigation.goBack()
  }

  return {
    productionType, setProductionType, quantity, setQuantity, unit, setUnit,
    date, setDate, animalId, setAnimalId, groupName, setGroupName,
    notes, setNotes, logMode, setLogMode, animals, loading, submit,
  }
}
```

### `CreateProductionLogScreen.tsx`
- **Production type selector**: Row of chips (Eggs, Milk, Fiber, Honey, Meat, Other)
- **Quantity** (NumberInput) + **Unit** display (auto-set from type, editable)
- **Date** (DatePicker, defaults to today)
- **Log mode toggle**: "Per Animal" / "Per Group"
  - Per Animal: DropdownSelect from user's animals
  - Per Group: TextInput for flock/herd name (e.g., "Laying Hens", "Dairy Goats")
- **Notes** (optional)
- Save button

```typescript
export default interface IProductionService {
  fetchProductionLogsByUser(userId: string): Promise<ProductionLog[]>
  fetchProductionLogsByAnimal(userId: string, animalId: string): Promise<ProductionLog[]>

```

### `ProductionService.ts`
- `fetchProductionLogsByUser`: query `productionLog` where `userId == uid` AND `admin.deleted == false`, order by `date` DESC
- `createProductionLog`: setDoc with auto-generated ID
- Registered in `Bootstrap.ts`

## Default Units

| Production Type | Default Unit |
|----------------|-------------|
| Eggs | count |
| Milk | gallons |
| Fiber | lbs |
| Honey | lbs |
| Meat | lbs |
| Other | count |

Users can override the unit per log entry. **Never mix units in aggregations** — only aggregate logs with matching units.

# Subscription Gating

- **Free tier**: No access to production logs. Attempting to create one triggers Pro paywall.
- **Pro/Farm tier**: Full access.

# Notes

- Production logging is a daily ritual — the quick-log entry makes it as fast as possible
- Per-animal OR per-group, never both — `animalId` and `groupName` are mutually exclusive
- Aggregation is client-side — all logs loaded into Zustand store, filtered and summed in the controller
- Trend visibility is important for farmers tracking seasonal patterns
- For MVP, a simple bar chart is sufficient — no need for a heavy charting library
- Units are flexible — while defaults are provided, users can log milk in liters instead of gallons
