## 2. Notification Functions

### `onCareEventDue.ts`
- Scheduled function (runs daily at 7:00 AM user timezone or UTC)
- Queries all careEvents where `completedDate == ''` AND `dueDate <= today`
- Groups by userId, sends push notification: "You have {count} care items due today"
- Creates notification data for in-app display

### `onBreedingDue.ts`
- Scheduled function (runs daily)
- Queries active breedingRecords where `expectedDueDate` is within 3 days
- Sends push notification: "{animalName} is due in {days} days"

### `onWithdrawalClear.ts`
- Scheduled function (runs daily)
- Queries healthRecords with `withdrawalPeriodDays > 0`
- Checks if withdrawal period ended today
- Sends notification: "Withdrawal period cleared for {animalName}"

## 3. Trigger Functions

### `onCareEventComplete.ts`
- Firestore trigger: `onUpdate` on `/careEvent/{eventId}`
- When `completedDate` changes from empty to a date AND `type == 'careRecurring'`:
  - Verify `createdNextRecurringEvent` is false
  - Create new careEvent with `dueDate = completedDate + cycle`
  - Set `createdNextRecurringEvent = true` on the completed event
- This is a safety net — the client should also create the next event, but the trigger ensures it happens even if the client fails

## 4. Scheduled Functions

### `dailyCareReminder.ts`
- Runs daily at 7:00 AM UTC
- Aggregates all care events due today across all users
- Sends batch push notifications via FCM
- Requires user's FCM token stored in `/user/{userId}/device/{deviceId}.tokenId`

## 5. `index.ts` — Export all functions

```typescript
export { dailyCareReminder } from './scheduled/dailyCareReminder'
export { onCareEventComplete } from './triggers/onCareEventComplete'
```

# Notes

- Keep functions minimal for MVP — only build what's absolutely needed
- Notifications require the user's FCM token stored in `/user/{userId}/device/{deviceId}`
- Use `firebase-admin` for server-side Firestore operations
- All functions should include error handling and logging
- Do NOT build: payments processing (RevenueCat handles this), analytics aggregation, AI features
- The `onCareEventComplete` trigger is a safety net — the client also creates the next recurring event to avoid latency
- Scheduled functions use Firebase's built-in scheduler (no external cron needed)
