# Goal

Build the Home tab (Today Dashboard) — the primary screen users see after login. This is the single screen that drives daily app opens. It aggregates overdue care, due-today care, active withdrawal alerts, active breeding countdowns, and recent activity into one actionable view.

## Home Screen

### `HomeController.ts`
```typescript
export function useHomeController(navigation) {
  const { user } = useUserStore()
  const { careEvents, fetchCareEvents, getOverdueEvents, getDueTodayEvents } = useCareStore()
  const { fetchActiveBreedings, breedingRecords } = useBreedingStore()
  const { getAllActiveWithdrawals, healthRecords } = useHealthStore()
  const [refreshing, setRefreshing] = useState(false)

  useFocusEffect(useCallback(() => {
    fetchCareEvents()
    fetchActiveBreedings()
    getAllActiveWithdrawals()
  }, []))

  const overdueEvents = getOverdueEvents()
  const dueTodayEvents = getDueTodayEvents()
  const activeBreedings = breedingRecords.filter(r => r.status === 'active')
  const activeWithdrawals = getActiveWithdrawals(healthRecords) // from WithdrawalUtility

  const onRefresh = async () => {
    setRefreshing(true)
    await Promise.all([fetchCareEvents(), fetchActiveBreedings(), getAllActiveWithdrawals()])
    setRefreshing(false)
  }

  const onCareEventPress = (eventId: string) => navigation.navigate('CareEventDetail', { eventId })
  const onAnimalPress = (animalId: string) => navigation.navigate('AnimalDetail', { animalId })
  const onBreedingPress = (recordId: string) => navigation.navigate('BreedingRecordDetail', { recordId })

  return {
    user, overdueEvents, dueTodayEvents, activeWithdrawals, activeBreedings,
    refreshing, onRefresh, onCareEventPress, onAnimalPress, onBreedingPress,
  }
}
```

### `HomeScreen.tsx`
- **Header row**: User avatar (top-left, opens drawer) + app title "Homestead" centered
- **Greeting**: "Good morning, {firstName}" (time-of-day aware: morning/afternoon/evening)
- **Sections in scroll order**:

1. **Overdue Care** (red section, only shown if items exist):
   - Section header: "Overdue" with count badge
   - List of `CareEventCard` components with red left border
   - Each card: event name, animal name, days overdue

2. **Due Today** (amber section, only shown if items exist):
   - Section header: "Due Today" with count badge
   - List of `CareEventCard` with amber left border
   - Each card: event name, animal name

3. **Active Withdrawal Alerts** (red, only shown if items exist):
   - Section header: "Active Withdrawals" with red badge
   - List of `DashboardAlertCard` with type='withdrawal'
   - Each card: animal name, medication name, withdrawal type (meat/milk/eggs), days remaining

4. **Breeding Countdowns** (shown if any active breedings):
   - Section header: "Upcoming Births"
   - List of `BreedingCard` (compact variant) with progress bar
   - Each card: dam name, days remaining, progress percent

5. **Recent Activity** (always shown):
   - Last 10 actions across all collections (care completions, health records, breeding updates)
   - Simple timeline items: icon + description + timestamp

- **Pull-to-refresh**: `RefreshControl` on the ScrollView
- **Empty state**: If no alerts, show "All caught up! Your animals are doing great." with a friendly icon
- **No FAB on this screen** — actions happen from other tabs

## Section Components

### `OverdueCareSection.tsx`
- Props: `events: CareEvent[]`, `onEventPress`
- Renders section header + list of care event cards with overdue styling
- Sorted by dueDate ascending (most overdue first)

### `DueTodaySection.tsx`
- Props: `events: CareEvent[]`, `onEventPress`
- Same pattern, amber styling

### `WithdrawalAlertSection.tsx`
- Props: `withdrawals: WithdrawalResult[]`, `onAnimalPress`
- Each item shows: animal name, medication, type (meat/milk/eggs), days remaining, red badge
- Sorted by days remaining ascending (most urgent first)

### `BreedingCountdownSection.tsx`
- Props: `breedings: BreedingRecord[]`, `onBreedingPress`
- Each item shows: dam name, expected due date, progress bar, days remaining
- Uses `calculateGestation()` from GestationUtility
- Label: "gestation" for mammals, "incubation" for poultry

### `RecentActivitySection.tsx`
- Props: `activities: ActivityItem[]`
- Simple timeline with icon + text + relative time
- Max 10 items

# Notes

- The Today Dashboard is read-only — it surfaces data from other domains
- Queries are cross-animal: care events, health records, breeding records all queried by userId
- Active withdrawals are computed client-side from health records (compare date + withdrawalPeriodDays to today)
- The dashboard should feel like a morning farm checklist — "here's what needs attention today"
- If all sections are empty, the app still feels useful with the greeting and empty state
- Performance: limit queries to necessary data only (e.g., only incomplete care events, only active breedings)
