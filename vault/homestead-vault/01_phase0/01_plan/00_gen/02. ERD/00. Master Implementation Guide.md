# Goal

Master guide for implementing the Homestead app using the generated plan documents. Defines the execution order, what to include in each prompt, validation checkpoints between layers, and rules for the LLM to follow.

# How to Use This Guide

Each implementation step below is a **single LLM prompt session**. For each step:

1. Copy the listed documents into the prompt context
2. Include the prompt instruction provided
3. Validate the output against the checkpoint criteria before moving to the next step
4. If validation fails, fix issues in the same session before proceeding

**Do not skip layers.** Each layer depends on the output of previous layers.

# Rules for the LLM

Include these rules in **every prompt session**:

```
## Rules
1. Follow the document instructions exactly. Do not invent features not described.
2. Use the exact file paths and naming conventions specified.
3. Every schema must include `admin: AdminObject`.
4. Use Uniwind (Tailwind CSS for React Native, drop-in NativeWind replacement) className for all styling. Do NOT use react-native-paper.
5. Use Zustand for state. Do NOT use rxjs, Redux, or Context.
6. Follow the Controller + Screen pattern: Controller is a custom hook, Screen is pure UI.
7. Services go through Bootstrap.ts — never import Firebase/MMKV directly in features.
8. All Firestore queries filter `admin.deleted == false` unless explicitly stated otherwise.
9. Include userId on all animal, care, health, breeding, production, note, and weight documents.
10. Do not add comments or documentation unless the document specifies it.
11. TypeScript strict mode. No `any` types unless absolutely necessary.
12. Import statements at the top of every file.
13. Dates stored as ISO 8601 strings. Use date-fns for all date calculations.
14. Subscription gating enforced client-side with RevenueCat entitlements.
```
