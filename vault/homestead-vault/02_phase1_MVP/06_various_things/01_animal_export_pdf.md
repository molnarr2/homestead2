# Animal Export & PDF Generation

## Summary

Add an export icon to the AnimalDetailHeader that opens a BottomSheet modal with four export options. Each option generates a professional PDF document from the animal's data using `react-native-html-to-pdf` and presents it via `react-native-share` for saving, printing, or sending. No backend changes required — all PDF generation happens on-device from data already in Zustand stores.

## Current Behavior

The AnimalDetailHeader (`apps/mobile/src/feature/animal/component/AnimalDetailHeader.tsx`) shows a back arrow and a pencil (edit) icon in the top bar. The screen has no export or share functionality. All animal-related data is already loaded into Zustand stores by the time the user views the detail screen: `animalStore`, `careStore`, `healthStore`, `breedingStore`, `noteStore`, `weightStore`, `groupStore`.

`react-native-share` (v12.2.4) is already installed but unused. No PDF library exists in the project.

## Desired Behavior

A new icon (material-design-icons `file-export-outline`) appears in the AnimalDetailHeader between the back arrow and the edit pencil. Tapping it opens a BottomSheet modal with four export buttons:

1. **Complete Animal Record** — PDF with every piece of data logged for this animal: general info, health records, care events, breeding records, production logs, weight history, notes, withdrawal status, and group memberships.

2. **Health & Care Summary** — PDF with general info + health care records + care events only. Designed for vet visits or sharing medical history.

3. **Breeding & Lineage Report** — PDF with general info + breeding records (with birth outcomes) + lineage (sire/dam names). Useful for breed registries, sales, or tracking bloodlines.

4. **Sale & Transfer Sheet** — PDF with general info + current weight + health summary (vaccination status, active withdrawals) + registration number + photo. A one-page document for buyers at livestock sales or farm-to-farm transfers.

After PDF generation, `react-native-share` opens the native share sheet so the user can save to Files, AirDrop, email, print, etc.

## PDF Document Design

Each PDF is an HTML document converted to PDF on-device. Professional styling with:

- Header: app name, homestead name, generation date
- Animal photo (if available, embedded as base64)
- Sections with clear headings, tables, and consistent formatting
- Footer: page numbers, "Generated by Homestead App"

### Data Sections

**General Info:** Name, type, breed, gender, birthday/age, color, registration #, state, purchase price, current weight, notes.

**Health Records:** Table sorted by date descending. Columns: Date, Type (vaccination/medication/deworming/vet visit/illness/injury), Name, Provider, Dosage (if applicable), Route, Notes, Cost. Withdrawal info inline.

**Care Events:** Table sorted by due date. Columns: Name, Type (single/recurring), Cycle, Due Date, Completed Date, Contact, Notes.

**Breeding Records:** Table sorted by breeding date. Columns: Breeding Date, Method, Sire Name, Status, Expected Due Date, Birth Date, Born Alive, Stillborn, Dam Condition, Complications, Offspring.

**Production Logs:** Table sorted by date. Columns: Date, Type, Quantity, Unit, Notes.

**Weight History:** Table sorted by date. Columns: Date, Weight, Unit, Body Condition Score, Change from Previous, Notes.

**Notes:** List sorted by date. Each entry: Date, Tags, Text.

**Active Withdrawals:** Table. Columns: Medication, Type (meat/milk/eggs/all), End Date, Days Remaining, Status.

**Lineage:** Sire name and ID, Dam name and ID (looked up from animalStore by sireId/damId).

## New Dependency

`react-native-html-to-pdf` — generates PDF files from HTML strings on-device. Supports iOS and Android. No native module linking issues on RN 0.76+.

Alternative considered: `react-native-print` — only prints, doesn't save to file. `react-native-html-to-pdf` is better because it produces a file path that can be passed to `react-native-share`.

## Data Access Audit

**No new database reads.** All data comes from Zustand stores that are already populated via `onSnapshot` subscriptions established at app boot. The export service reads from store getters — zero Firestore queries on the export path.

**Sire/Dam lookup:** The animal's `sireId` and `damId` are resolved by scanning `animalStore.animals` in memory. No additional query needed.

**Photo embedding:** The animal's `photoUrl` is fetched as a base64 string via `fetch()` at PDF generation time. This is a single HTTP request to Firebase Storage, acceptable latency for a user-initiated export action.

**Round trip count:** 0 Firestore reads. 1 optional HTTP request (photo). PDF generation is CPU-only.

## Data Migration

None. No schema changes. No new Firestore documents. Export is read-only.

## Risk

**PDF generation performance.** An animal with hundreds of health records could produce a large HTML document. Mitigation: generation runs on the JS thread but is a single synchronous string concatenation followed by a native PDF render call. The native PDF engine handles pagination. Show a loading spinner during generation.

**Photo embedding latency.** Fetching the photo URL and converting to base64 adds 1-3 seconds on slow connections. Mitigation: if the fetch fails or times out (5s), generate the PDF without the photo rather than failing the entire export.

**HTML rendering differences.** iOS and Android PDF engines render HTML slightly differently. Mitigation: use simple HTML/CSS only — tables, headings, paragraphs. No flexbox, no grid, no web fonts. Test on both platforms.
